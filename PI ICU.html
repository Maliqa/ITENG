PI System 2018 SP2 OPC DA Directed Activity Bab 2 — PI Interface Management
Section 2.7

Apa itu PI Interface Configuration Utility (PI ICU)?

PI Interface Configuration Utility (ICU) adalah Graphical User Interface (GUI) yang digunakan oleh PI System Administrator untuk membuat dan mengkonfigurasi batch file serta service dari PI Interface.

PI ICU hanya bisa mengkonfigurasi batch file dan service yang terinstal di komputer yang sama (tidak bisa remote ke komputer lain).

📝

Module Database (MDB): Setelah PI ICU digunakan untuk mengkonfigurasi batch file, isi file tersebut ditulis ke database di Data Archive yang disebut "Module Database" (MDB). Ini memungkinkan recovery konfigurasi PI Interface. Namun jika batch file diedit secara manual, PI ICU akan menampilkan warning.

💡

Konfigurasi PI Interface melalui PI ICU (bukan edit manual) adalah best practice agar konfigurasi tersimpan di MDB dan bisa di-recover kapan saja.

Section 2.10

Metodologi Instalasi PI Interface

Setiap kali PI Interface baru dipasang untuk pengumpulan data, ikuti metodologi berikut secara berurutan:

1

Pilih PI Interface

Tentukan jenis PI Interface sesuai data source (OPC DA, OPC UA, Modbus, dll.)

2

Pilih Lokasi Instalasi

Tentukan arsitektur: apakah interface di-install di server terpisah atau di PI Server.

3

Install PI Interface, PI ICU & PI API

Install PIICU_x.x.xx.xx_.exe, OPCInt_ReadOnly.exe, dan PI API for Windows Integrated Security.

4

Validasi Koneksi ke Data Archive

Verifikasi konektivitas jaringan (ping, port TCP 5450) dan test PI SDK + PI API dari interface node.

5

Validasi Data di OPC Server

Gunakan PI OPC Client Tool untuk memastikan data tersedia di OPC Server sebelum konfigurasi interface.

6

Konfigurasi Security

Buat PI Mapping untuk service account PI Interface di Data Archive.

7

Buat & Konfigurasi Instance Interface

Di PI ICU: buat instance baru, set PointSource, Interface ID, Scan Class, dan OPC Server target.

8

Buat PI Points

Buat PI Tag untuk setiap OPC Item yang akan dikumpulkan datanya, via SMT atau PI Builder.

9

Konfigurasi Buffering

Aktifkan PI Buffer Subsystem agar data tidak hilang saat jaringan atau Data Archive down.

10

Buat Health Points

Buat UniInt Health Points (Heartbeat, Device Status, IO Rate) untuk monitoring kesehatan interface.

✅

Step 1–8 adalah langkah dasar untuk mulai mengumpulkan data. Step 9–10 diperlukan untuk memastikan pengumpulan data yang reliable di environment produksi.

Section 2.12.8

Tipe PI Point untuk PI Interface for OPC DA

Ada empat cara berbeda untuk mendefinisikan PI Points untuk PI Interface for OPC DA, tergantung kebutuhan pengambilan data:

TipeLocation3Cara KerjaUse Case
Polled 0 Interface polling OPC Server secara periodik sesuai frekuensi scan class Data yang butuh interval konsisten
Advise 1 OPC Server mengirim nilai baru ke interface ketika ada perubahan (read on change) Paling efisien — kurangi traffic jaringan, tidak ada duplikat nilai
Event (Trigger) 0 Interface minta nilai baru ke OPC Server ketika nilai trigger PI Point berubah Data yang harus diambil berdasarkan event tertentu
Output 2 Interface menulis nilai dari PI Point ke data source (bukan untuk pengumpulan data) Write-back ke sistem kontrol (tidak tersedia di versi Read-Only)
⭐

Metode Advise sering kali paling efisien dan performan. Gunakan ini sebagai default untuk sebagian besar PI Points.

Aturan Atribut PI Point untuk OPC DA:

AtributFungsi
Location1Interface ID — menghubungkan PI Point ke instance interface tertentu
Location3Tipe PI Point: 0 = Polled/Event, 1 = Advise, 2 = Output
Location4Nomor scan class yang digunakan
PointSourceHarus match dengan Point Source dari instance interface
InstrumentTagOPC Item ID di OPC Server — harus persis sama, case-sensitive!
⚠️

Aturan penting: Hanya Advise points yang boleh masuk ke scan class 1. PI Points dengan tipe berbeda tidak boleh berada di scan class yang sama.

Section 2.15 — Directed Activity

Konfigurasi PI Buffer Subsystem

PI Buffer Subsystem adalah aplikasi OSIsoft yang menyimpan data sementara di interface node saat koneksi ke Data Archive terputus. Analoginya seperti reservoir — data terakumulasi saat "valve" ke Data Archive tertutup, lalu otomatis dikirim saat koneksi pulih.

⚠️

PI Buffer Subsystem wajib dikonfigurasi di environment produksi. Tanpanya, data akan hilang setiap kali Data Archive atau jaringan down.

Section 2.16 — Directed Activity

UniInt Health Points — Monitor Kesehatan Interface

UniInt Health Points adalah PI Points khusus yang mengumpulkan informasi tentang kesehatan PI Interface. Dibuat dari PI ICU di menu UniInt → Health Points. OSIsoft merekomendasikan minimal 4 Health Points berikut:

Heartbeat
Menunjukkan apakah interface sedang berjalan. Nilai berputar dari 1–15 selama interface aktif. Berhenti update jika interface mati atau deadlock. Tidak menunjukkan apakah interface terhubung ke data source.
Device Status
Informasi komunikasi antara interface dan data source. Normal = GOOD. Jika ada masalah, berisi kode status: contoh 95 | Device(s) in error berarti tidak bisa komunikasi ke OPC Server.
IO Rate
Menghitung semua nilai PI Point yang dikirim ke Data Archive. Jika nilai berhenti update, berarti interface berhenti mengumpulkan data.
Scan Class Scans Skipped
Menghitung "skipped scans" — scan yang tidak dilakukan sebelum jadwal scan berikutnya tiba. Periode pelaporan default: 8 jam. Nilai tinggi mengindikasikan beban berlebihan.

Manage Interface yang Sudah Ada dengan PI ICU

Directed Activity Familiar dengan PI ICU — Edit Konfigurasi Interface Existing § 2.8

Membiasakan diri dengan PI ICU dengan memuat instance yang sudah ada dan mengedit konfigurasinya untuk memahami dampak perubahan.

  • 1

    Login dan Jalankan PI ICU

    Login ke PIINT02. Jalankan program "PI Interface Configuration Utility".

  • 2

    Pilih Interface Instance

    Dari dropdown "Interface", pilih opcint_ReadOnly1. Catat nilai Point Source yang ada di tab General.

  • 3

    Verifikasi Data di SMT

    Di PISRV01, buka SMT → Data → Current Values, load semua PI Points dengan Point Source tersebut. Klik "Start Updating" dan perhatikan kecepatan update data.

  • 4

    Edit Scan Class

    Di PI ICU, klik kanan pada scan class #1 → ubah ke 00:00:01 (1 detik) → klik "Apply" → restart interface dengan tombol Restart di pojok kiri atas.

  • 5

    Amati Perubahan di SMT

    Kembali ke SMT — data seharusnya update lebih cepat sekarang (setiap 1 detik vs sebelumnya).

  • 6

    Ubah Point Source (Eksperimen)

    Kembali ke PI ICU, ubah nilai Point Source ke nilai yang berbeda → klik "Apply" → restart interface.

  • 7

    Amati Dampak Perubahan Point Source

    Kembali ke SMT. PI Points yang sudah ada menggunakan Point Source lama — karena Point Source tidak match lagi dengan interface, data akan berhenti masuk. Ini menunjukkan betapa kritisnya kecocokan Point Source.

  • 8

    Undo Perubahan

    Kembali ke PI ICU, kembalikan Point Source ke nilai semula → Apply → restart interface. Data seharusnya kembali normal.
