Prerequisites

Komponen yang Harus Ada

Pastikan semua ini sudah terinstall dan running sebelum mulai

VC-8000 MPS Rack — hardware vibration monitor, sudah terpasang dan powered on
Setpoint PI Adapter (π) — software interface, gratis, download dari BKV website
Setpoint CMS Software — client display app, gratis, terinstall di komputer CMS
PI Server (Data Archive + AF) — sudah running, semua services aktif
PI AF Client — terinstall di komputer yang sama dengan CMS Display
SQL Server Express (min 2012) — untuk AF database (PIFD), gratis
Network: koneksi antara VC-8000, PI Adapter node, dan PI Server
ℹ️
Lisensi: CMS Display dan PI Adapter gratis dari Brüel & Kjær Vibro. Yang berbayar hanya OSIsoft PI System license (per-server/per-tag) dan Setpoint-to-PI Adapter license (per PI Server instance).
01
Setup

Setup PI Adapter

Konfigurasi koneksi antara MPS rack dan PI Server

STEP 01
Buka PI Adapter
Jalankan sebagai Administrator untuk akses penuh.
# Start Menu → Setpoint - PI Adapter Setup # Atau klik kanan → Run as Administrator Tampilan awal: ┌──────────────────────────────────────────┐ │ Action │ Status │ Rack IP │ CMS-DB │ │ Record │ Stopped │ 192.168.0.1│ TRC1_45 │ └──────────────────────────────────────────┘ Tombol: Add | Edit | Delete | Check Connections
STEP 02
Tambah Rack Baru
Klik Add untuk mendaftarkan VC-8000 rack baru ke PI Adapter.
Klik: Add → Isi field berikut: Rack IP Address : 192.168.0.1 ← IP VC-8000 rack CMS-XC Database : TRC1_45 ← nama CMS database PI Server : PISRVCISDEMO ← hostname PI Server AF Root Path : Setpoint\SiteA\Turbine AF Rack Alias : STG1_11 ← nama rack di AF tree → OK
STEP 03
Aktifkan PI Backfilling (Tab Advanced)
Fitur ini memastikan data yang hilang saat network putus akan otomatis di-recover saat reconnect.
Klik Edit → Tab: Advanced ☑ PI Backfilling ← centang ini PI Server : PISRVCISDEMO AF Root Path : Setpoint\SiteA\Turbine AF Rack Alias: STG1_11 → OK Note: jika gap > 3 hari, perlu prosedur manual khusus
⚠️
Backfilling hanya bekerja untuk PI System — tidak berfungsi untuk mode CMS-XC (offline mode).
STEP 04
Check Connections — Verifikasi Semua Terhubung
Gunakan ini sebelum start, saat commissioning, atau saat ada masalah koneksi.
Klik: Tools → Check Connections # Hasil yang diharapkan (semua ✅): ✅ All Connections Passed ✅ CMS-XC ✅ Service ✅ AF [PISRVCISDEMO] ✅ PI [PISRVCISDEMO] ✅ VC-8000 Rack [192.168.0.1] # Jika ada yang ❌: ❌ Rack [192.168.0.1] → Connected to 192.168.0.1:8001. Name: (kosong) → Cek IP rack & pastikan VC-8000 menyala
02
Setup

Konfigurasi VC-8000 MPS

Enable CMS mode di hardware rack sebelum PI Adapter bisa collect data

STEP 01
Enable CMS di VC-8000
Buka MPS Setup Software dari komputer yang terhubung ke rack.
MPS Setup Software → Rack Settings CMS (SD) Enabled : ✅ ← simpan ke SD card (max 32 GB) CMS (HD) Enabled : ✅ ← simpan ke internal SSD (max 256 GB) CMS (DAC) Enabled : ✅ ← stream ke PI via network CMS (DAC) Rack Name : STG1_11 CMS (DAC) IP Address : 192.168.0.1
STEP 02
Pastikan Module UMM/SAM Sudah CM-Enabled
Setiap modul di rack harus diaktifkan secara individual untuk bisa streaming CMS data.
MPS Setup → Module Settings → per Slot Slot 2 - SAM: ☑ CMS ☑ HD32 ☐ Standard Display Slot 3 - UMM: ☑ CMS Slot 4 - UMM: ☑ CMS # Jika module belum CM-enabled → Enable Hardware # PI license based on IP — gunakan static IP untuk DAC port!
⚠️
PI license berbasis IP address. Pastikan VC-8000 menggunakan static IP untuk port DAC/CMS — jangan DHCP!
03
Setup

Start Data Collection

Mulai recording dan verifikasi data masuk ke PI Archive

STEP 01
Start Recording di PI Adapter
PI Adapter → pilih rack → Klik: Record (▶) # Status berubah: Stopped → Starting → Recording ✅ # Indikator di header: 🟢 TRC_1 (5.0.1227) ✅ XC ← connected & recording 🔴 setpoint2 (5.1.2001) ✅ XC ← stopped
STEP 02
Yang Terjadi Otomatis Saat Pertama Connect
PI Adapter akan otomatis melakukan seluruh proses ini tanpa perlu input manual:
Proses Otomatis PI Adapter: 1. Baca konfigurasi MPS dari VC-8000 rack 2. Auto-create PI Tags di Data Archive 3. Auto-create AF Database structure di PI AF 4. Mulai collect data → kirim ke PI Archive # Struktur AF yang terbentuk otomatis: Setpoint Instrument (rack) └─ Customer Site A └─ Unit 501 ├─ A1 ├─ B10 └─ B11 ├─ Bearing Radial Vibration 1 └─ Radial Vibration 2
STEP 03
Format Nama PI Tag yang Dibuat Otomatis
Setiap channel menghasilkan multiple PI tags. Radial Vibration = 17 tags per channel.
// Tag Name Format
RackName • Slot • Channel • Measurement • Type

STG1_11.4.1.Direct.0 ← Direct vibration (Float32, mil/µm)
STG1_11.4.1.Gap.0 ← Gap voltage (Float32, mil/µm)
STG1_11.4.1.nXAmplitude.1 ← 1X Amplitude
STG1_11.4.1.nXAmplitude.2 ← 2X Amplitude
STG1_11.4.1.nXPhase.1 ← 1X Phase (deg)
STG1_11.4.1.SyncWaveform.99 ← Waveform data (Float32)
STG1_11.4.1.Direct.0.Quality ← Quality/alarm status (Int32)

Penggunaan

SETPOINT CMS Display

Aplikasi utama untuk analisis vibration — orbit, FFT, waveform, Bode

BUKA
Buka CMS Software & Navigasi
# Start Menu → Setpoint CMS # Login dengan Windows credential Panel Kiri (Navigation Tree): Assets └─ Setpoint Instrument (rack) └─ Customer Site A └─ Unit → Bearing → Channel Cara buka plot: 1. Klik bearing/channel di navigation tree 2. Pilih measurement (1X Amplitude, Direct, dll) 3. Pilih plot type dari toolbar 4. Set time range: * - 8h sampai * 5. Klik: Refresh / Update
〰️
Time Waveform
Raw sinyal vibration vs waktu. Untuk deteksi impak dan bentuk gelombang.
📈
Spectrum (FFT)
Frequency domain. Identifikasi komponen 1X, 2X, harmonic, bearing defect freq.
🔄
Orbit Plot
Gerakan shaft X vs Y. Deteksi misalignment, rub, dan shaft instability.
📉
Bode Plot
Amplitude & phase vs RPM. Identifikasi critical speed saat startup/shutdown.
🧭
Polar Plot
Phase vector diagram. Monitor perubahan phase selama operasi.
🌊
Waterfall / Cascade
Spectrum vs waktu 3D. Lihat evolusi frekuensi dari waktu ke waktu.
📊
Trend
Nilai measurement over time. Monitoring kondisi jangka panjang.
⚠️
Waveform data TIDAK bisa dilihat di PI Vision! Untuk orbit plot, spectrum, FFT — wajib pakai Setpoint CMS Display. PI Vision hanya untuk static data (amplitude, direct, phase).

Penggunaan

Upload Offline Data / Backfilling

Recovery data CMS yang collected offline atau saat jaringan putus

UPLOAD
Upload CMS Database Offline ke PI
Digunakan ketika data dikumpulkan tanpa koneksi PI (mode XC/offline).
PI Adapter → Options → PI System Options → Klik: Upload (Import CMS Data into PI System) # Dialog Upload CMS Database: CMS Database File : C:\...\data-file.cms PI Server : PISRVCISDEMO AF Rack Alias : STG1_11 → OK # Response sistem: "PI System is ready to archive the upload range" ✅ → Data akan di-upload "PI System was not running during this period" ⚠️ → Admin perlu buat archive baru dulu
BACKFILL
Auto Backfilling via PI Buffer Subsystem
Jika jaringan putus sementara, PI Adapter otomatis backfill data dari CMS-HD SAM.
# Cara kerja auto backfilling: 1. Network putus → data tersimpan di CMS-HD (SSD internal) 2. Network reconnect → PI Adapter deteksi gap 3. Data otomatis di-push ke PI Archive (backfill) 4. Gap terisi → data kontinu di PI Vision ✅ # Jika gap lebih dari 3 hari: ⚠️ Perlu prosedur manual → Refer ke CMS manual untuk instruksi lengkap → Hubungi Brüel & Kjær Vibro support
✅
Backfilling hanya bekerja jika PI Backfilling diaktifkan di tab Advanced PI Adapter (lihat Section 01 Step 03).

Administrasi

Rename PI Tags

Kustomisasi nama tag sesuai konvensi naming perusahaan tanpa reconfig rack

PI BUILDER
Rename via PI Builder (Excel Add-in)
PI Adapter menggunakan kolom instrumenttag (bukan name) untuk mapping — jadi nama bisa diubah bebas.
Buka Excel → Tab: PI Builder → PI Points → Load tags dengan prefix STG1_11* # Kolom yang boleh diubah: Name → ganti sesuai naming convention Description → update deskripsi yang lebih jelas # Kolom yang JANGAN diubah: instrumenttag → ini yang dipakai PI Adapter untuk mapping rack! # Contoh rename: Before: STG1_11.4.1.Direct.0 After : Plant.VibMotor01.BrgDE.Direct → Publish ke PI Server ✅ # PI Adapter akan tetap bisa collect data karena # mapping berdasarkan instrumenttag, bukan tag name
ℹ️
AF Elements juga bisa dipindah-pindah di tree tanpa perlu reconfig rack. Extended properties menyimpan Setpoint rack identifier secara terpisah.
