// Langkah-Langkah Konfigurasi
01
WAJIB PERTAMA · PI System Explorer
Buat AF Database Kosong
Buat hanya container database — jangan buat element atau hierarchy di dalamnya. PI Adapter akan otomatis mengisi seluruh hierarchy saat Record pertama kali dijalankan.
# Start Menu → PI System Explorer → Login → Connect to: PISRVCISDEMO File → New Database Name : Setpoint Description : SETPOINT CMS Asset Database → OK # ⛔ STOP — JANGAN buat element apapun di dalamnya! # Biarkan database kosong seperti ini: Setpoint ← hanya database, tidak ada child element → Check-in (Ctrl+K) ← wajib simpan ke server # Catat nama database untuk diisi di PI Adapter: AF Database Name: Setpoint
⚠️
Jangan buat element atau hierarchy di dalam database! PI Adapter yang akan auto-create berdasarkan konfigurasi MPS rack. Jika sudah terlanjur dibuat, hapus dulu sebelum lanjut.
02
Hardware · VC-8000 MPS Setup
Konfigurasi VC-8000 & Enable CMS
Aktifkan CMS mode di rack VC-8000 dan pastikan IP address static — karena PI license terikat ke IP address.
# MPS Setup Software → Rack Settings CMS (SD) Enabled : ✅ ← buffer SD card (max 32 GB) CMS (HD) Enabled : ✅ ← buffer SSD internal (max 256 GB) CMS (DAC) Enabled : ✅ ← stream data ke PI via network Rack Name : STG1_11 IP Address : 192.168.0.1 ← HARUS STATIC IP! # Enable CMS per modul di setiap slot: Slot 2 - SAM: ☑ CMS ☑ HD32 Slot 3 - UMM: ☑ CMS Slot 4 - UMM: ☑ CMS
⚠️
PI license berbasis IP address. Gunakan static IP untuk DAC/CMS port. Jangan DHCP karena IP bisa berubah dan menyebabkan license error.
03
Software · PI Adapter Setup
Konfigurasi PI Adapter
Install PI Adapter lalu tambah rack. AF Root Path dikosongkan (blank) — PI Adapter akan otomatis resolve ke root AF Database.
# Start Menu → Setpoint - PI Adapter Setup (Run as Admin) Klik: Add Rack IP Address : 192.168.0.1 CMS-XC Database : STG1_45 PI Server : PISRVCISDEMO AF Root Path : (kosong / blank — jangan diisi apapun!) AF Rack Alias : STG1_11 ← nama rack di AF (bebas) Tab: Advanced ☑ PI Backfilling ← aktifkan untuk auto-recovery data PI Server : PISRVCISDEMO AF Rack Alias: STG1_11 → OK
💡
AF Root Path dibiarkan kosong (blank) — PI Adapter otomatis resolve ke root AF Database yang sudah dikonfigurasi di PI Server. Tidak perlu diisi nama database atau path apapun.
04
Verifikasi · Check Connections
Cek Semua Koneksi
Wajib semua ✅ sebelum Record. Jika ada ❌ selesaikan dulu.
Tools → Check Connections ✅ All Connections Passed ✅ CMS-XC ✅ Service ✅ AF [PISRVCISDEMO] ✅ PI [PISRVCISDEMO] ✅ VC-8000 Rack [192.168.0.1] # Jika gagal: ❌ AF [PISRVCISDEMO] → DB belum di-Check-in / piaflink down ❌ Rack [192.168.0.1] → IP salah atau VC-8000 mati ❌ PI [PISRVCISDEMO] → PI services tidak running
🚫
Jangan klik Record jika masih ada ❌ Failed. Selesaikan semua error terlebih dahulu.
05
START · Data Collection
Klik Record — Auto-Create Semua
Setelah AF Database kosong ada dan semua koneksi ✅, klik Record. PI Adapter akan otomatis membuat seluruh hierarchy, tags, dan attribute mapping.
PI Adapter → pilih rack → Klik: Record (▶) Stopped → Starting → Recording ✅ # Yang terjadi OTOMATIS saat pertama kali Record: 1. Connect ke VC-8000 → baca konfigurasi rack 2. Auto-create PI Tags di Data Archive Format: RackName.Slot.Channel.Measurement.Index Contoh: STG1_11.4.1.nXAmplitude.1 3. Auto-create AF Elements & Hierarchy di database Setpoint → Setpoint Instrument → Site → Unit → Bearing 4. Auto-create AF Attributes + mapping ke PI Tags 5. Auto-create Element Templates per tipe channel 6. Data mulai mengalir ke PI Archive ✅ # Verifikasi di PI System Explorer setelah Record: Setpoint └─ Setpoint Instrument (rack) ← auto dibuat └─ Customer Site A ← auto dibuat └─ Unit 501 ← auto dibuat └─ B11 ← auto dibuat ├─ Bearing Radial Vibration 1 └─ Radial Vibration 2
✅
PI Adapter tidak override perubahan manual yang sudah lo buat di AF setelah hierarchy terbentuk. Lo bebas rename tag atau pindah element setelah struktur terbentuk.
06
Verifikasi · PI System Explorer
Pastikan AF Hierarchy Sudah Terbuat
Buka PI System Explorer dan verifikasi bahwa PI Adapter sudah berhasil membuat seluruh hierarchy element dan attributes secara otomatis.
# Start Menu → PI System Explorer → Connect to: PISRVCISDEMO # Pilih AF Database: Setpoint # ✅ Hierarchy yang harus sudah terbuat otomatis: Setpoint └─ Setpoint Instrument (rack) └─ STG1_11 ← nama sesuai AF Rack Alias ├─ Bearing Radial Vibration 1 │ ├─ Attribute: Direct → STG1_11.4.1.Direct.0 │ ├─ Attribute: 1X Amplitude → STG1_11.4.1.nXAmplitude.1 │ ├─ Attribute: 2X Amplitude → STG1_11.4.1.nXAmplitude.2 │ └─ Attribute: Quality → STG1_11.4.1.Direct.0.Quality └─ Bearing Radial Vibration 2 └─ ... # Cara verifikasi attribute sudah mapping ke PI Tag: Klik element (mis: Bearing Radial Vibration 1) → Tab: Attributes → Klik attribute (mis: Direct) → Kolom Value harus tampil: angka real-time ✅ → Bukan: No Data atau Shutdown
⚠️
Jika hierarchy belum terbuat setelah Record dijalankan: cek PI Adapter status apakah benar-benar Recording (bukan hanya Starting), dan pastikan koneksi ke VC-8000 rack tidak putus saat proses init pertama kali.
ℹ️
Jika AF baru saja dibuat, tunggu ~5 menit sebelum hierarchy muncul di navigation tree PI Vision. PI Vision membutuhkan waktu untuk index AF yang baru.
07
Verifikasi · PI SMT
Verifikasi PI Tags di PI SMT
Konfirmasi PI Tags sudah terbuat di Data Archive dan nilai sudah masuk secara real-time.
# Start Menu → PI System Management Tools (Run as Admin) Data → Current Values → Tag Search (ikon teropong) Tag Name: STG1_11* → Search # Harusnya muncul list tags (17 per Radial Vibration channel): STG1_11.4.1.Direct.0 ← Direct vibration STG1_11.4.1.Gap.0 ← Gap voltage STG1_11.4.1.nXAmplitude.1 ← 1X Amplitude STG1_11.4.1.nXAmplitude.2 ← 2X Amplitude STG1_11.4.1.nXPhase.1 ← 1X Phase STG1_11.4.1.SyncWaveform.99 ← Sync Waveform STG1_11.4.1.Direct.0.Quality ← Quality/Alarm status ... # Klik "Start Updating" → kolom Value harus berubah real-time # Jika nilai = Shutdown atau No Data → cek interface status di PI ICU → Klik Start Updating → nilai berubah real-time ✅
✅
Jika Step 06 (AF Hierarchy) dan Step 07 (PI Tags) keduanya sudah terkonfirmasi — sistem SETPOINT CMS + PI Adapter sudah berjalan penuh dan siap digunakan.

Buka & Navigasi CMS Display

# Start Menu → Setpoint CMS → Login Windows credential Navigation Panel (kiri): Assets └─ Setpoint Instrument (rack) └─ Customer Site A └─ Unit 501 → Bearing → Channel Cara buka plot: 1. // Langkah Membuat Display PI Vision
P1
Home Page
Buat Display Baru
Dari home page klik New Display. Opsional: buat folder dulu untuk organisasi display.
Home Page → Klik: + New Display → Display kosong terbuka, Design mode aktif otomatis # Buat folder dulu (opsional tapi disarankan): Home → Folder icon (kiri) → New Folder Name: SETPOINT Vibration # Save display ke folder: Ctrl+S → Save As Name : STG1_Turbine_Overview Folder: SETPOINT Vibration ☑ Inherit permissions from folder → Save
P2
Assets Pane · Cari Data
Cari PI Tags atau AF Attributes
Ada dua cara mencari data di Assets Pane.
# CARA 1: Search by Name Assets Pane → kotak Search → ketik: STG1_11* ← wildcard, tampilkan semua tag rack ini *Direct* ← semua tag yang mengandung "Direct" nXAmplitude ← prefix match → Tekan Enter # CARA 2: Navigation Tree (lebih visual, AF-based) Assets Pane → pilih: Setpoint (AF Database) → Drill down: Setpoint → Setpoint Instrument → Site A → Unit 501 → STG1_11 → Bearing Radial Vibration 1 → (Attributes pane: Direct, 1XAmplitude, dll) # Tips: • Jangan pakai tanda kutip pada search term • * selalu di-append otomatis di akhir query • AF baru butuh ~5 menit untuk muncul di search
P3
Symbol · Trend
Tambah Trend (Grafik Time-Series)
Trend menampilkan nilai data vs waktu. Support multi-trace (beberapa tag dalam satu trend).
# Tambah Trend: Symbol Gallery → Trend 📈 → Drag tag ke canvas # Tambah trace ke trend yang sudah ada: → Drag tag lain ke atas trend yang sudah ada # Drag multiple tags sekaligus: → Tahan Ctrl + klik beberapa tag → Drag ke canvas # Konfigurasi (klik kanan → Configure Trend): Value Scales: Scale Type : Multiple scales ← skala terpisah tiap trace Scale Range : Database limits ← pakai Zero/Span dari PI point Time Range: Start/End : Display time range ← sinkron dengan Time Bar Trace Options: Color : pilih warna per trace Style : Line / Data Markers / Scatter Plot Weight : ketebalan garis (1-5)
P4
Symbol · Value
Tambah Value (Angka Real-time)
Menampilkan nilai current satu tag. Update setiap 5 detik. Bisa dikombinasikan dengan Multi-state.
Symbol Gallery → Value 🔢 → Drag tag ke canvas # Format (klik kanan → Format Value): Style: Format : Number → Decimal Places: 2 Value Color: pilih warna Visibility: ☑ Label → Custom: 1X Amplitude DE ☑ Units → tampilkan satuan (mm/s, µm) ☑ Timestamp → tampilkan waktu update terakhir ☑ Value → tampilkan angka
P5
Symbol · Gauge
Tambah Gauge (Indikator Visual)
Tampilan visual seperti speedometer. 3 tipe: Vertical, Horizontal, Radial. Cocok untuk monitoring getaran.
Symbol Gallery → Gauge (V/H/Radial) → Drag tag ke canvas # Format (klik kanan → Format Gauge): Scale Range: ○ Database limits ← pakai Zero & Span dari PI Point ● Custom limits: Min : 0 Max : 25 ← mm/s (ISO 10816) Visibility: ☑ Label → Vibration RMS ☑ Value → angka current ☑ Units → satuan ☑ Scale → tampilkan skala
💡
Radial gauge paling cocok untuk vibration monitoring — mudah dibaca operator secara sekilas mirip speedometer.
P6
Symbol · Multi-State
Konfigurasi Multi-State (Visual Alarm)
Ubah warna symbol otomatis berdasarkan range nilai. Cocok untuk Quality tag SETPOINT — VALID/ALERT/DANGER.
# Langkah: 1. Buat Value symbol dengan Quality tag: Drag: STG1_11.4.1.Direct.0.Quality 2. Klik kanan → Add Multi-State 3. Property: ● Fill ← background color berubah 4. Set kondisi numerik + warna: Max = 1 → Gray Label: NO DATA Max = 2 → Green Label: VALID Max = 8 → Yellow Label: ALERT Max = 32 → Red Label: DANGER Max = 64 → Dark Red Label: NOT OK Bad Data → Dark Gray 5. Untuk blink saat Danger: State DANGER → ☑ Blink
🎯
Multi-state juga bisa diterapkan ke Gauge, Table, Bar Chart, dan Shape/Rectangle — tidak hanya Value symbol. Berguna untuk membuat mimik diagram alarm.
P7
Design Mode · Layout
Atur Layout & Elemen Tambahan
Move, resize, tambah text label, shape, dan gambar untuk display yang informatif.
# Move & Resize: drag symbol / drag corner handle # Multi-select : Shift+klik / drag area # Group : Ctrl+G # Text Label: Toolbar → Text (T) → klik canvas → ketik label → klik kanan → Format Text: Font Size 14, Bold # Shape (untuk grouping visual): Toolbar → Draw Shape → Rectangle → klik kanan → Format: Fill Transparent, Outline color # Upload gambar / logo P&ID: Toolbar → Upload Image → pilih PNG/JPG
