Pengenalan

PI AF itu seperti apa?

📁 Bayangkan PI AF seperti "folder pintar" di komputer kamu.

PI Data Archive menyimpan ribuan angka sensor setiap detik — tapi angka saja tidak cukup. Kamu perlu tahu: angka ini dari mesin apa? bagian mana? unit berapa? Nah, PI AF inilah yang menjawab pertanyaan itu. AF mengorganisir semua data PI Tags ke dalam struktur hierarki yang mencerminkan aset fisik pabrik — dari level plant, unit, sampai komponen terkecil.

Perbandingan

PI Data Archive vs PI Asset Framework

🗃️ PI Data Archive
📊 Menyimpan data time-series (nilai sensor tiap detik)
🏷️ Data diakses lewat nama tag seperti A1_SY0410C
❓ Tidak tahu tag itu dari mesin atau sensor apa
📂 Seperti hard disk — menyimpan semua file tapi tidak terstruktur
🔢 Di WW: total 6.000 tag tersedia
🏗️ PI Asset Framework (AF)
🌳 Mengorganisir tag ke dalam hierarki aset pabrik
🔗 Tag dihubungkan ke elemen: Turbine → Shaft Vibration → 1X
✅ Setiap atribut punya makna konteks yang jelas
🗂️ Seperti Windows Explorer — file yang sama tapi terorganisir
⚙️ Di WW: database "Vibration" untuk data VC-8000

Konsep Inti

4 Komponen Utama PI AF

🗄️

AF Database

Wadah utama yang menampung semua element dan template. Di WW, nama database-nya "Vibration" — khusus menyimpan data CMS dari VC-8000.

🧩

Element

Representasi digital dari aset fisik. Contoh: UNIT #1, Turbine, Generator. Mirip seperti folder yang mewakili sebuah mesin nyata di lapangan.

📋

Template

Cetakan (blueprint) untuk element. Kalau ada 10 unit turbin, cukup buat 1 template, terus tinggal di-instantiate 10 kali. Hemat waktu konfigurasi.

🔗

Attribute

Data point spesifik dari sebuah element — dihubungkan langsung ke PI Tag. Contoh: attribute "1X Direct" → PI Tag UNIT#1.3.1.Direct.0.

📏

UOM (Unit of Measure)

Satuan pengukuran yang terdaftar di AF Server. Contoh: µm untuk vibrasi, mm/s untuk velocity, RPM untuk kecepatan.

🔍

AF Search / Query

Fitur pencarian element berbasis hierarki. Memudahkan PI Vision menampilkan data berdasarkan konteks aset, bukan hanya nama tag mentah.

Integrasi

Bagaimana SETPOINT Connector Bekerja dengan AF?

1

VC-8000 mengukur vibrasi

Hardware VC-8000 (IP: 172.18.1.10, port 8001) terus-menerus mengukur vibrasi shaft turbine, generator, bearing, dan Hot Well Pump dari Rotor Kit unit 1.

2

SETPOINT Connector membaca data

Service SetpointConnector.MainService berjalan otomatis di IDWWDW25SCADA02, membaca data dari VC-8000 secara real-time setiap siklus (default ~8 detik untuk waveform).

3

Menulis ke PI Data Archive

Data vibrasi (Direct, 1X Amplitude, 1X Phase, Gap, dsb.) ditulis langsung ke PI Tags di server 172.18.1.11 menggunakan write access via PIUSER dan PIPOINT table.

4

Membuat/update AF Element & Attribute

Sekaligus, Connector juga menulis ke AF Server (IDWWDW25SCADA02). Ia membuat element hierarki (UNIT#1 → Turbine → Shaft Vibration 1X) dan menghubungkan setiap attribute ke PI Tag yang sesuai.

5

PI Vision membaca dari AF

Ketika operator membuka dashboard PI Vision, PI Vision mengambil data lewat AF — sehingga yang tampil bukan tag mentah, tapi konteks yang bermakna: "Turbine Shaft 1X Direct Vibration: 42 µm".

⚠️

Setiap PI Tag vibrasi juga punya pasangan .Quality tag (tipe Int32, Point Source: SETPOINT) yang mencatat status kualitas data. Kalau Quality = 0 artinya data valid; nilai lain menunjukkan kondisi khusus sensor.

Checklist Operasional

Yang Perlu Dicek Terkait AF

  • SETPOINT Connector berjalan di status Running di Windows Services pada IDWWDW25SCADA02
  • Check Connection SETPOINT: semua item All Connections Passed (lihat gambar verifikasi di project report)
  • AF Database "Vibration" di IDWWDW25SCADA02 berstatus Found & Permissions OK
  • PI Server (172.18.1.11) dapat diakses dari AF Server — AFSDK terinstall minimum versi 2.08
  • PI Tags VC-8000 tidak ada yang missing (target: 0 missing dari 224/256 tags)
  • PI Vision Asset Servers menampilkan database "Vibration" dengan checkbox tercentang
  • Service account STARENERGYID\svc_pi memiliki permission Administrators + PIWorld di AF Server
  • Backfill aktif — data historis VC-8000 bisa masuk ke PI Archive saat koneksi kembali normal
Troubleshooting

Masalah Umum AF & Solusinya

🔴

PI Vision tidak melihat asset hierarchy

Cek Configuration → Asset Servers di PI Vision Admin. Pastikan IDWWDW25SCADA02 connected dan database "Vibration" dicentang.

🟡

Data VC-8000 tidak update di AF

Cek status SETPOINT Connector di Windows Services. Lakukan Check Connection — lihat apakah ada item merah. Biasanya problem koneksi ke VC-8000 atau PI Server.

🔵

Element AF hilang atau kosong

Jalankan ulang SETPOINT Connector setelah pastikan koneksi ke VC-8000 dan PI/AF normal. Connector akan re-create element yang hilang otomatis.

🟢

Tag missing saat Check Connection

Di SETPOINT Check Connection, jika "Can create tags: True" maka Connector bisa membuat tag baru. Jalankan Connector dan biarkan berjalan beberapa menit — tag akan terbuat otomatis.

🎯 Ringkasan: Kenapa PI AF Penting?

PI AF adalah yang membuat PIMS Wayang Windu bisa menjawab pertanyaan: "Gimana kondisi vibrasi shaft Turbine Unit 1 sekarang?" — bukan sekadar menampilkan angka tag UNIT#1.3.1.Direct.0 = 42. Dengan AF, data punya konteks, makna, dan hierarki yang bisa langsung dipahami oleh operator dan engineer tanpa harus hafal ratusan nama tag.
