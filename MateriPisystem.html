Sesi 1

Pengenalan PI System & Arsitektur

08:00 – 09:00  ·  60 menit
Materi

📦 Apa itu Industrial Historian?

Software yang merekam data proses industri (suhu, tekanan, flow rate, level) secara real-time dan menyimpannya dalam time-series database yang dioptimalkan untuk data sensor — bukan database relasional biasa.

🏢 Sejarah Singkat

OSIsoft PI System berdiri sejak 1980 dan menjadi standar de-facto industri O&G. Diakuisisi AVEVA tahun 2021, lalu menjadi bagian Schneider Electric. Digunakan di lebih dari 1.000 perusahaan energi dan manufaktur global.

🔷 5 Komponen Utama

PI Data Archive — database time-series
PI AF — asset framework & konteks
PI Vision — visualisasi real-time
PI Web API — REST interface
PI Interface / Connector — koneksi data

🏭 Kenapa PI System?

Kompresi data efisien (hingga 10x lebih kecil), integrasi mudah ke DCS/SCADA/OPC, analytics historis, dan menjadi standar di industri O&G, Petrokimia, Power, dan Manufaktur.

Sesi 2

PI Data Archive & PI Tag (PI Point)

09:00 – 10:30  ·  90 menit  ·  Coffee break 10:15
Anatomy PI Tag

Apa itu PI Tag / PI Point?

PI Tag adalah representasi satu sinyal atau sensor dalam PI System. Setiap tag adalah satu "kolom" dalam time-series database. Kalau pabrik punya 20.000 sensor, berarti ada 20.000 PI Tags.

Contoh Nama Tag

PUMP-01.VIBRATION.MOTOR_RMS
HX-02.TEMPERATURE.INLET
COMP-01.PRESSURE.DISCHARGE

Konvensi: Equipment.Parameter.Location

Atribut Utama PI Tag
AtributFungsiContoh
TagNameNama unik tag di dalam PI ServerPUMP01.VIB.RMS
PointTypeTipe data nilai yang disimpanFloat32, Int32, String, Digital
PointSourceIdentifier interface yang menulis data ke tag iniO (OPC), M (Manual), C (Connector)
InstrumentTagNama tag di OPC server / DCS sumberAI1001, FIC_101.PV
EngUnitSatuan engineeringmm/s, °C, bar, m³/h
DescriptorDeskripsi singkat tagMotor 01 Vibration RMS
Zero / SpanBatas bawah dan atas nilai sensor0 / 100
Compression & Exception

Exception Deviation

Data hanya dikirim dari interface ke PI Data Archive kalau nilainya berubah melebihi threshold yang ditentukan. Tujuannya: menghemat bandwidth jaringan antara interface node dan PI Server.

Compression Deviation

Data hanya disimpan permanen ke archive kalau berubah secara signifikan (di luar deadband). Tujuannya: menghemat storage di PI Data Archive, bisa hingga 10x lebih efisien vs menyimpan setiap sample.

Compression On/Off

Default: Compression = On. Nonaktifkan hanya untuk data kritis yang wajib tersimpan setiap detik tanpa ada nilai yang dilewati, misalnya data safety atau billing meter.

Snapshot vs Archive

Snapshot = nilai terkini (real-time) dari tag, disimpan di memory. Archive = data historis yang sudah lolos compression, disimpan permanen ke disk.

🎤 Kalimat Presenter

"Oke sekarang kita masuk ke inti PI System — yaitu PI Tag atau PI Point. PI Tag itu ibarat satu kolom di spreadsheet, tapi khusus time-series. Satu tag sama dengan satu sinyal sensor."

"Perhatikan contoh nama tagnya: PUMP-01.VIBRATION.MOTOR_RMS. Ini artinya tag untuk data vibrasi RMS motor pada Pump nomor 01. Naming convention yang konsisten itu sangat penting — kalau tagnya ribuan, naming yang buruk akan menyulitkan semua orang."

"Sekarang soal compression. Bayangin sensor suhu yang nilainya stabil di 65°C sepanjang hari — apakah kita perlu simpan nilai yang sama itu setiap detik? Tentu tidak. Nah, di sinilah fungsi Compression — PI hanya menyimpan data kalau ada perubahan yang signifikan. Hasilnya: storage kita bisa hemat sampai 10 kali lipat."

"Exception Deviation itu 'filter' pertama sebelum data masuk ke PI Server lewat jaringan. Compression Deviation itu 'filter' kedua sebelum data ditulis permanen ke disk. Keduanya bekerja bersama untuk membuat PI System sangat efisien."

Koneksi Data: OPC & PI Interface

10:30 – 12:00  ·  90 menit
OPC DA vs OPC UA
AspekOPC DA (Classic)OPC UA
KepanjanganOLE for Process Control — Data AccessUnified Architecture
TeknologiDCOM/COM — Windows onlyTCP/IP — platform independent
KeamananTerbatas, port dinamis (sulit firewall)Enkripsi & autentikasi built-in
PenggunaanLegacy DCS — masih sangat banyak digunakanModern, IIoT, cloud-ready
PI InterfacePI Interface for OPC DA (OPCInt.exe)PI Connector for OPC UA
Cocok untukSistem DCS lama (ABB, Yokogawa, Emerson)Sistem baru, SCADA modern
Alur Konfigurasi PI Interface for OPC DA
1. Buka PI ICU
PI Interface Configuration Utility — tool untuk manage semua PI Interface
2. Pilih OPCInt.exe
Locate executable di folder PI installation, register sebagai Windows Service
3. Set OPC Server
Isi nama host & OPC Server ProgID, misal: Matrikon.OPC.Simulation.1
4. Set PointSource
Tentukan PointSource identifier, misal: O untuk OPC. PI tag yang ber-PointSource=O akan dibaca oleh interface ini
5. Start Service
Jalankan interface sebagai Windows Service. Cek PI SMT — data seharusnya mulai mengalir ke PI Tags

InstrumentTag vs Location

Untuk PI Interface for OPC DA, field InstrumentTag diisi dengan Item ID dari OPC Server (misal: Random.Int1). Atribut Location1–4 digunakan untuk konfigurasi interface lama.

Matrikon OPC Simulation Server

OPC server gratis untuk keperluan lab dan testing. Mensimulasikan berbagai tipe sinyal: random, ramp, sawtooth. Sangat cocok untuk belajar tanpa perlu DCS fisik.

🎤 Kalimat Presenter

"Sekarang pertanyaannya: bagaimana data dari DCS atau sensor di lapangan bisa sampai ke PI Data Archive? Jawabannya adalah lewat PI Interface atau PI Connector."

"OPC itu analoginya seperti USB — satu standar konektor universal untuk semua device industri. Tanpa OPC, setiap vendor DCS punya protokol yang berbeda-beda dan kita harus buat driver custom untuk masing-masing. Dengan OPC, PI Interface cukup 'bicara OPC' dan bisa connect ke hampir semua sistem DCS."

"Sekarang ada dua versi: OPC DA yang sudah lama dan masih banyak dipakai di sistem DCS legacy, dan OPC UA yang lebih modern dan aman. Di proyek-proyek existing seperti yang berbasis ABB atau Yokogawa lama, kemungkinan besar kalian akan ketemu OPC DA."

"Untuk lab kita hari ini, kita pakai Matrikon OPC Simulation Server sebagai pengganti DCS. Ini tools yang sering banget dipakai engineer PI untuk testing dan belajar tanpa perlu hardware fisik."

Sesi 4

PI Asset Framework (PI AF)

13:00 – 14:30  ·  90 menit
PI DA vs PI AF

PI Data Archive — "Gudang Data"

Menyimpan data time-series. Tidak tahu konteks: tidak tahu tag itu milik equipment mana, di unit mana, di plant mana. Hanya tahu: nama tag, waktu, dan nilai.

PI Asset Framework — "Peta Aset"

Menambahkan struktur dan konteks: Plant › Unit › Equipment. Setiap Equipment adalah sebuah Element, dan tag PI-nya menjadi Attribute dari element tersebut.

Konsep Kunci PI AF
KonsepPenjelasanAnalogi
DatabaseWadah utama semua aset dalam satu PI AF ServerSatu perusahaan / satu plant
ElementRepresentasi satu aset fisik (pump, compressor, vessel)Satu baris di daftar equipment
AttributeProperti dari element — bisa link ke PI Tag, atau nilai statisKolom di tabel equipment
TemplateBlueprint element — buat satu template, apply ke semua equipment sejenisClass / blueprint di OOP
AnalysisKalkulasi berbasis attribute: rata-rata, alarm, prediksiRumus di Excel, tapi real-time
Contoh Hierarchy PI AF
🏭 FSO Platform (Database)
├── ⚙️ Compression Train A (Element — Unit)
│ ├── 🔩 Pump-01 (Element — Equipment)
│ │ ├── Vibration.RMS → PUMP-01.VIB.RMS
│ │ ├── Temperature.Motor → PUMP-01.TEMP.MOTOR
│ │ └── Speed.RPM → PUMP-01.SPEED
│ └── 🔩 Pump-02 (Element — Equipment)
│ ├── Vibration.RMS → PUMP-02.VIB.RMS
│ └── Temperature.Motor → PUMP-02.TEMP.MOTOR
└── ⚙️ Compression Train B (Element — Unit)
🎤 Kalimat Presenter

"Kalau PI Data Archive itu 'gudang data'-nya — dia menyimpan semua nilai sensor — maka PI AF itu adalah 'peta'-nya. Tanpa PI AF, kita hanya punya ribuan nama tag yang membingungkan, tanpa tahu tag itu milik equipment apa, di unit mana, dan di plant mana."

"Bayangkan kalian dapat daftar 20.000 nama tag tanpa konteks apapun — pusing kan? PI AF menyelesaikan itu dengan membuat hierarchy: Plant → Unit → Equipment → Parameter. Sekarang operator bisa langsung klik 'Pump-01' dan langsung lihat semua datanya: vibrasi, suhu, RPM — semuanya."

"Fitur paling powerful di PI AF adalah Template. Kalau kalian punya 50 pompa yang sejenis, cukup buat satu template Pump, lalu apply ke 50 equipment sekaligus. Efisiensinya luar biasa."

"Di proyek nyata seperti di platform offshore, PI AF ini yang jadi 'backbone' — semua 20.000 tag diorganisir dalam hierarchy yang mencerminkan struktur fisik plant. Operator bisa navigasi dari platform level sampai ke sensor individual dalam beberapa klik."

Sesi 5

PI Vision — Visualisasi Data Real-Time

14:30 – 15:30  ·  60 menit  ·  Coffee break 15:15
Fitur Utama PI Vision

📈 Trend Chart

Tampilkan data historis dan real-time dalam grafik garis. Bisa overlay beberapa tag sekaligus untuk perbandingan, dengan zoom dan pan timeline yang mudah.

🔢 Value Display

Angka live dari PI Tag yang ter-update otomatis. Bisa dikonfigurasi warna berdasarkan threshold: hijau = normal, kuning = warning, merah = alarm.

⏱️ Gauge / Dial

Representasi analog untuk nilai seperti RPM, tekanan, dan suhu. Lebih intuitif bagi operator yang terbiasa dengan panel kontrol fisik.

🔗 Koneksi ke PI AF

Display bisa di-link ke Element di PI AF. Operator cukup pilih equipment mana yang ingin dilihat, dan display otomatis menampilkan semua attribute-nya.

📋 Event Frame

Tampilkan event tertentu seperti startup, shutdown, atau anomaly berdasarkan kondisi yang sudah didefinisikan di PI AF Analysis.

🔐 User Management

Role-based access: Read-only untuk operator, Edit untuk engineer, Admin untuk PI administrator. Integrasi dengan Windows AD.

Cara Buat Display Pertama
1

Buka PI Vision di Browser

Akses http://[server-name]/PIVision — tidak perlu install software di client, cukup browser modern.

2

Klik "New Display"

Pilih layout kosong. Klik tombol Add Symbol di toolbar, pilih tipe symbol: Trend, Value, Gauge, dll.

3

Assign PI Tag atau AF Attribute

Drag & drop tag dari PI Tag Search, atau navigasi ke AF Element lalu pilih attribute yang ingin ditampilkan.

4

Konfigurasi Threshold & Warna

Double-click symbol → Format panel → atur color zones sesuai batas normal/warning/alarm dari ISO 10816 atau SIL requirement.

5

Save & Share

Save display, atur permission siapa saja yang bisa akses. Display bisa dibuka di PC, tablet, bahkan smartphone.

Sesi 6

Hands-On Lab

15:30 – 17:00  ·  90 menit + Q&A
Langkah Lab
01

Verifikasi Stack PI System

Buka PI SMT → cek semua service UP: PI Data Archive, PI AF Server, PI Vision. Test koneksi PI Web API via browser: https://[server]/piwebapi/

02

Buat PI Tags Manual via PI SMT

PI SMT → Tag Search → New Tag. Buat 4 tag:
Motor01.Vibration.RMS · Motor01.Temperature
Motor02.Vibration.RMS · Motor02.Temperature
PointType: Float32 | EngUnit: mm/s atau °C

03

Jalankan Python RMS Simulator

Jalankan pi_rms_simulator.py di terminal. Script ini mensimulasikan data vibrasi Motor01–04 sesuai threshold ISO 10816 (OK / Warning / Alarm) dan mengirim data via PI Web API setiap 2 detik. Verifikasi di PI SMT: Tag Search → lihat Current Value.

04

Buat Hierarchy PI AF

Buka PI System Explorer → New Database → Buat Element CompressorTrain → child element Motor01 → tambah Attribute: Vibration.RMS (link ke PI Tag Motor01.Vibration.RMS), Temperature.

05

Build Display di PI Vision

Buka http://[server]/PIVision → New Display → Tambah Trend dengan tag Motor01.Vibration.RMS → Tambah Value display untuk Motor01.Temperature → Set threshold warna → Save display.

Penutup

Key Takeaways

Ringkasan akhir training
  • 01 PI System adalah platform historian industri standar de-facto di O&G dan Petrokimia. Bukan sekadar database — ini adalah infrastruktur data operasional yang menghubungkan sensor lapangan ke decision maker.
  • 02 PI Data Archive menyimpan time-series data dengan kompresi efisien. PI AF menambahkan konteks aset dan hierarchy. Keduanya bekerja bersama — Archive adalah gudang datanya, AF adalah peta-nya.
  • 03 OPC DA dan OPC UA adalah protokol standar yang menjembatani DCS/sensor ke PI Data Archive lewat PI Interface atau PI Connector. Pahami perbedaannya karena di lapangan kalian akan ketemu keduanya.
  • 04 PI Vision adalah web-based dashboard yang mengubah data menjadi insight visual real-time. Tidak perlu install apapun di sisi client — bisa diakses dari mana saja.
  • 05 PI Web API membuka PI System ke ekosistem modern: Python, Node.js, web dashboard, hingga AI/ML pipeline. Ini yang membuat PI System relevan di era Industry 4.0.
