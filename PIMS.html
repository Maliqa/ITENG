🎤 OPENING Pembuka — pengenalan sistem secara umum

"Bapak/Ibu, izinkan kami menjelaskan arsitektur sistem Plant Information Management System (PIMS) yang telah kami implementasikan di Wayang Windu. Secara keseluruhan, sistem ini terdiri dari tiga lapisan utama — mulai dari pengumpulan data di lapangan, penyimpanan dan pengolahan di server, hingga visualisasi yang dapat diakses oleh tim operasional kapan saja melalui web browser."

📡 LAYER 1 Data Sources — sumber data dari lapangan

"Di lapisan pertama, terdapat dua sumber data yang mengalirkan informasi ke dalam sistem PIMS secara real-time."

⚙️ Sumber Data 1 — ABB DCS via OPC DA

"Sumber pertama adalah ABB DCS System yang terhubung menggunakan protokol OPC DA — yaitu standar komunikasi data industri yang umum digunakan pada sistem kontrol generasi sebelumnya. Dari DCS ini, kami berhasil mengintegrasikan 215 tags yang mencakup parameter proses penting seperti suhu bearing turbin, tekanan wellhead sumur produksi, flow rate brine, kecepatan turbin, arus motor cooling tower fan, hingga level kondensor. Semua data ini masuk ke PI Server melalui PI Interface yang berjalan sebagai Windows Service di server."

📡 Sumber Data 2 — VC-8000 SETPOINT Vibration

"Sumber kedua adalah VC-8000 SETPOINT dari Brüel & Kjær Vibro, yaitu sistem monitoring vibrasi yang terpasang langsung di unit turbin-generator. Hardware ini mengumpulkan 224 tags data vibrasi yang sangat detail dan presisi — mencakup vibrasi shaft 1X dan 1Y di sisi turbin, vibrasi shaft 2X dan 2Y di sisi generator, thrust bearing axial, HWP-A, serta phase trigger. Data dari VC-8000 ini dikirimkan ke PI Server menggunakan software SETPOINT Connector yang terinstall di Server 2."

🖥️ LAYER 2 PI Servers — penyimpanan & pengelolaan data

"Di lapisan kedua, kami menggunakan dua server Dell PowerEdge R760XS yang beroperasi di jaringan OT internal Wayang Windu dengan subnet 172.18.1.x. Kedua server ini saling terhubung dan bekerja sebagai satu ekosistem PI System yang terintegrasi."

🖥️ Server 1 — IDWWDW25SCADA01 (172.18.1.11) — PI Data Archive

"Server pertama berperan sebagai PI Data Archive — ini adalah inti atau jantung dari seluruh sistem PIMS. Semua data sensor dari ABB DCS maupun VC-8000 tersimpan di sini secara real-time dalam format time-series yang sangat efisien. Server ini menjalankan PI Data Archive versi 3.4.460.29, dengan kapasitas archive 10 GB per file yang tersimpan di drive D, dan dilengkapi dengan backup otomatis setiap hari pukul 03:15 pagi secara incremental. Lisensi yang terpasang mendukung hingga 6.000 PI Tags, dan saat ini sudah aktif sekitar 439 tags. Keamanan akses diatur melalui PI Mappings dan PI Trusts berbasis Active Directory perusahaan."

🖥️ Server 2 — IDWWDW25SCADA02 (172.18.1.12) — PI Vision + AF

"Server kedua berperan sebagai PI Vision dan PI Asset Framework. Server ini yang mengelola visualisasi data dan struktur aset. PI Asset Framework (AF) mengorganisir seluruh data dalam hierarchy yang mencerminkan struktur fisik plant — mulai dari unit turbin, masing-masing bearing, hingga setiap parameter vibrasi yang dimonitor. Database aset yang kami bangun diberi nama 'Vibration' yang merepresentasikan seluruh monitoring vibrasi di Unit 1. Selain itu, server ini juga menjalankan SETPOINT Connector untuk menerima data langsung dari hardware VC-8000."

🔗 Koneksi Antar Server

"Kedua server ini saling berkomunikasi melalui jaringan internal. Ketika operator membuka PI Vision di browser, Server 2 secara otomatis mengambil data historis maupun real-time dari Server 1 melalui port TCP 5450 — port standar komunikasi PI System. Proses ini berlangsung otomatis dan transparan bagi pengguna."

👷 LAYER 3 End User Access — akses operator & engineer

"Di lapisan ketiga adalah akses untuk seluruh pengguna — baik operator, engineer, maupun manajemen. Tim Bapak/Ibu dapat membuka dashboard PI Vision langsung melalui web browser tanpa perlu menginstall software apapun di komputer client."

🌐 Cara Akses

"Alamat aksesnya adalah https://172.18.1.12/PIVision. Sistem keamanannya menggunakan Windows Integrated Authentication berbasis Active Directory domain STARENERGYID, sehingga user cukup login sekali dengan akun domain masing-masing. Hak akses diatur berdasarkan role — operator mendapat akses view untuk monitoring, engineer mendapat akses edit untuk membuat atau memodifikasi display, dan administrator memiliki akses penuh ke seluruh konfigurasi sistem."

✅ CLOSING Penutup & serah terima sistem

"Secara keseluruhan, sistem ini telah berhasil mengintegrasikan 439 tags aktif dari dua sumber data yang berbeda — yaitu DCS proses dan monitoring vibrasi — ke dalam satu platform terpadu yang dapat diakses secara real-time. Dengan arsitektur ini, tim Bapak/Ibu dapat memonitor kondisi operasional plant secara komprehensif, baik dari ruang kontrol maupun dari workstation di kantor, selama terhubung ke jaringan internal Wayang Windu."

"Kami juga telah menyiapkan dokumentasi lengkap sebagai bagian dari deliverable proyek ini — mencakup as-built configuration, prosedur backup & restore, panduan troubleshooting, serta user guide untuk operator dan administrator — untuk memastikan sistem ini dapat dioperasikan dan dimaintain secara mandiri oleh tim Bapak/Ibu ke depannya."

"Demikian penjelasan arsitektur sistem PIMS Wayang Windu dari kami. Apabila ada pertanyaan lebih lanjut mengenai konfigurasi, pengoperasian, maupun rencana pengembangan sistem ke depan, kami siap mendiskusikannya bersama. Terima kasih."

01
Persiapan Server & Network
Sebelum install apapun — pastikan fondasi infrastruktur siap
1.1
Rack & Power — Pastikan server Dell R760XS sudah terpasang di rack, power supply redundant aktif, dan UPS terhubung.
1.2
OS & Network — Install Windows Server 2025 di kedua server. Set hostname: IDWWDW25SCADA01 dan IDWWDW25SCADA02. Set IP static: 172.18.1.11 dan 172.18.1.12, gateway 172.18.1.254.
1.3
Join Domain — Join kedua server ke domain STARENERGYID (Active Directory).
1.4
Test Konektivitas — Ping antar server, ping ke VC-8000 (172.18.1.10), dan ping ke ABB DCS. Pastikan semua reachable.
1.5
Firewall Rules — Buka port TCP 5450 (PI DA), TCP 443 (HTTPS PI Vision), TCP 8001 (VC-8000 CMS) di Windows Firewall dan network firewall ABB.
1.6
Service Accounts — Buat domain service account STARENERGYID\svc_pi di Active Directory. Password jangan expire. Ini akan dipakai oleh PI Interface, PI Buffer, dan SETPOINT Connector.
02
Install PI Data Archive (Server 1)
Core PIMS — semua data tersimpan di sini
2.1
Run Installer — Di IDWWDW25SCADA01, jalankan installer PI Data Archive 2023. Pilih drive D untuk archive path: D:\PI\arc.
2.2
Konfigurasi Archive — Set ukuran archive file: 10,240 MB. Buka SMT → Operation → Archives, verifikasi archive file sudah terbuat.
2.3
Set Backup Schedule — SMT → Operation → Backups → konfigurasi jadwal daily incremental pukul 03:15 AM.
2.4
Apply Lisensi — SMT → Operation → Licensing. Apply license file (Installation ID: 10287114, 6000 tags). Pastikan Data Streams Allowed: 6016.
2.5
Verifikasi Services — SMT → Operation → PI Services. Pastikan semua subsystem running: pisnapss, piarchss, pibufss, piupdmgr, dll.
03
Install PI AF + PI Vision (Server 2)
Konteks aset & visualisasi dashboard
3.1
Install SQL Express — Install SQL Server Express di IDWWDW25SCADA02. Ini dibutuhkan oleh PI Vision sebagai database internal.
3.2
Install PI AF Server — Jalankan installer PI Asset Framework Server. Set AF Server name = IDWWDW25SCADA02. Set SQL instance ke \SQLEXPRESS.
3.3
Install PI Vision — Jalankan installer PI Vision. Pilih SQL database name: PIVISION. IIS akan dikonfigurasi otomatis oleh installer.
3.4
Konfigurasi HTTPS — Di IIS Manager, pastikan binding PIVision menggunakan HTTPS port 443. HTTP redirect ke HTTPS.
3.5
Test PI Vision — Buka browser, akses https://172.18.1.12/PIVision/admin. Verifikasi halaman Overview terbuka dan SQL Server status hijau.
04
Konfigurasi Security & PI Mappings
Kontrol akses berbasis Active Directory
4.1
Buat PI Identities — SMT → Security → Identities, Users & Groups. Pastikan ada: piadmin, piadmins, PIWorld.
4.2
Buat PI Mappings — SMT → Security → Mappings & Trusts → tab Mappings. Tambahkan mapping: Everyone → piadmin, IDWWDW25SCADA01\Administrator → piadmin, STARENERGYID\svc_pi → piadmin. Set semua Enabled = True.
4.3
Buat PI Trusts — Tab Trusts. Tambahkan trust untuk: Proxy_12/1 (loopback), AFSERVER2, ABB OPC FES, BufferingTrust_AFSERVER (app: pibufss.exe), Client1, Client2.
4.4
PI Vision Security — Buka PI Vision Admin → Security → Identity. Set Identity AF Server: IDWWDW25SCADA02. Auth: Windows only.
4.5
Set User Access Levels — PI Vision Admin → Security → User Access Levels. Set: Administrators → Administrator, World → Explorer.
05
Install & Konfigurasi PI Interface for OPC DA
Koneksi ABB DCS → PI Data Archive (215 tags)
5.1
Install PI ICU & OPCInt — Di IDWWDW25SCADA01, install: PIICU_x.x.xx.exe lalu OPCInt_ReadOnly_x.x.x.exe lalu PIAPI-xxxx-Windows-Integrated-Security.exe. Semua Run as Administrator.
5.2
Validasi OPC Server — Jalankan PI OPC Client Tool. Connect ke localhost → pilih ABB OPC Server. Pastikan Server Status = RUNNING sebelum lanjut.
5.3
Buat Instance di PI ICU — Buka PI ICU → Create new Interface Instance from .EXE → browse ke OPCInt_ReadOnly.exe. Set host PI Server: IDWWDW25SCADA01. Set PointSource: ABB-OPC. Klik Add → OK.
5.4
Tab General — Set Interface ID = 1. Tambah scan class frequency sesuai kebutuhan (contoh: 00:00:05 untuk 5 detik).
5.5
Tab OPCInt — Set OPC Server Node Name = localhost (atau IP ABB DCS). Klik List Available Servers → pilih OPC Server ABB.
5.6
Tab Service — Log on as: STARENERGYID\svc_pi. Klik Create. Kemudian Save konfigurasi.
5.7
Start & Monitor — Klik View PI Message Log → klik Start. Pastikan muncul pesan: Connected to OPC Server dan OPC Server current state = RUNNING.
06
Buat PI Tags — ABB OPC DA (215 Tags)
Bulk import via PI Builder dari PI OPC Client Tool
6.1
Generate CSV — Buka PI OPC Client Tool → connect ke ABB OPC Server → Add Group → Browse OPC Server → Select All items → File → Save As. Set PointSource = ABB-OPC → Save sebagai .csv.
6.2
Edit CSV di Excel — Buka file CSV. Set kolom: location1=1, location2=0, location3=1 (Advise), location4=1 (scan class 1), location5=0. Rename kolom Tag sesuai naming convention (contoh: A1_TE0401).
6.3
Publish via PI Builder — Di Excel, klik tab PI Builder → Publish → Edit Mode: Create Only → OK. Tunggu hingga muncul "The requested action is complete."
6.4
Set EngUnit & Descriptor — Gunakan PI Builder → Find PI Points (filter: PointSource=ABB-OPC) → edit kolom engunits dan descriptor → Publish dengan Edit Mode: Edit Only.
6.5
Verifikasi — SMT → Data → Current Values → filter PointSource = ABB-OPC. Semua 215 tags harus menampilkan nilai real-time dalam 2 menit setelah interface berjalan.
07
Install & Konfigurasi SETPOINT Connector (VC-8000)
Koneksi VC-8000 vibration hardware → PI System (224 tags)
7.1
Install SETPOINT Connector — Di IDWWDW25SCADA02, install SETPOINT Connector Setup 2025 (v8.6.240.0). Run as Administrator.
7.2
Add Data Connection — Buka SETPOINT Connector → klik Add. Connection Type: VC-8000. IP Address: 172.18.1.10. Port: 8001. Klik OK.
7.3
Konfigurasi PI System — File → PI System. Set: PI Server = 172.18.1.11, AF Server = IDWWDW25SCADA02, AF Database = Vibration. Centang "Collect data" dan "Backfill into PI System".
7.4
Check Connections — Klik tombol Check Connections. Tunggu hasil. Semua item harus centang hijau — terutama: All Connections Passed, Rack\PI tag comparison: 224/224 Missing: 0.
7.5
Set Service Account — Windows Services → SetpointConnector.MainService → Properties → Log On → set akun STARENERGYID\svc_pi. Restart service.
7.6
Verifikasi Tags — SMT → Data → Current Values → filter PointSource = SETPOINT. Tag vibrasi (UNIT#1.3.1.Direct.0, dll) harus mendapat data real-time.
08
Buat PI Asset Framework (AF)
Hierarchy aset Unit 1 — Turbine, Generator, Bearing, HWP
8.1
Buat AF Database — Buka PI System Explorer (PSE) → klik kanan pada server IDWWDW25SCADA02 → New Database → nama: Vibration → Create.
8.2
Buat Element Templates — Di Library → Element Templates. Buat templates sesuai jenis sensor VC-8000: Setpoint.Element.Template.Bearing, HydroGenerator, Rack, Cylinder, dll. Setiap template berisi attribute sesuai channel (Direct, Gap, 1X Amplitude, 1X Phase, dll).
8.3
Buat Hierarchy Elements — Di Elements → buat struktur: Setpoint Racks → UNIT #1 → child elements: Turbine, Generator, Axial, HWP-A, Speed. Apply template yang sesuai ke masing-masing element.
8.4
Link PI Tags ke Attribute — Pada setiap attribute di element, set Data Reference = PI Point. Set tag name sesuai PI Tag yang sudah dibuat (contoh: UNIT#1.3.1.Direct.0).
8.5
Check In — Klik Check In untuk save semua perubahan ke AF Database. Verifikasi di PSE bahwa semua element dan attribute tersimpan.
09
Konfigurasi Buffering & Backup
Pastikan data tidak hilang saat network atau server down
9.1
Setup PI Buffer Subsystem — Buka PI ICU → Tools → Buffering → Yes → Continue with configuration. Pilih server IDWWDW25SCADA01 dan interface yang ada.
9.2
Set Service Account Buffer — Di wizard, klik Change → Use Windows account → masukkan STARENERGYID\svc_pi. Set PI Identity ke PI Interface & PI Buffers → Create.
9.3
Set Buffer Queue Location — Pilih drive terpisah dari OS (contoh: E:\OSIsoft\Buffering). Selesaikan wizard → Exit.
9.4
Aktifkan per Interface — Di PI ICU → tab General → centang opsi buffering untuk setiap interface. Pastikan dependency PIBufss otomatis ditambahkan.
9.5
Verifikasi Buffering — Buka Buffering Manager (Tools → Buffering). Pastikan: Global Buffering Status = No reported issues. Total events sent terus bertambah. Estimated capacity mencukupi.
9.6
Verifikasi Backup — SMT → Operation → Backups. Cek Backup History — pastikan status Success. Jalankan manual backup sekali untuk test: klik Start Backup.
10
PI Vision Display & Final Verification
Build dashboard dan verifikasi end-to-end sebelum serah terima
10.1
Link Data Server — PI Vision Admin → Configuration → Data Servers. Pastikan server IDWWDW25SCADA01 versi 3.4.460.29 sudah terdaftar dan status hijau. Klik Test Connection → Allowed.
10.2
Link Asset Server — Tab Asset Servers. Pastikan IDWWDW25SCADA02 terdaftar. Expand → centang database Vibration. Klik Save.
10.3
Buat Display Proses — Buka PI Vision → New Display. Tambahkan Trend untuk tag proses ABB (suhu, tekanan, flow). Tambahkan Value Display untuk parameter kritis. Set threshold warna.
10.4
Buat Display Vibrasi — New Display lagi. Tambahkan Trend untuk tag VC-8000 (contoh: UNIT#1.3.1.Direct.0, UNIT#1.3.1.nXAmplitude.1). Link ke AF Element UNIT#1 supaya bisa switch equipment.
10.5
Set Display Permission — Klik kanan display → Manage Security. Set siapa yang bisa view dan siapa yang bisa edit.
10.6
End-to-End Test — Login ke PI Vision dari komputer client menggunakan akun domain user biasa (bukan admin). Pastikan display terbuka, data real-time update, historical trend bisa dilihat.
10.7
Serah Terima — Siapkan dokumen Project Report (as-built config, tag list, network diagram, prosedur backup, user guide). Tandatangani Project Sign-off bersama representatif Star Energy. Sistem resmi dioperasikan oleh tim customer. ✅
