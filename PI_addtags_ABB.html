01 — Overview

Arsitektur Integrasi ABB → PI

Di Wayang Windu, ABB DCS (Distributed Control System) bertindak sebagai sumber data proses utama — mengumpulkan data dari ribuan instrumen di lapangan seperti sensor suhu, tekanan, flow, dan level. Data dari ABB DCS diekspos melalui OPC DA Server, kemudian dibaca oleh PI OPC DA Interface dan dikirim ke PI Data Archive.

🏭

ABB DCS

Controller utama proses di Wayang Windu. Mengekspos data via OPC DA Server yang berjalan di Engineering Workstation (EWS) atau OPC Server Node ABB.

🔌

PI OPC DA Interface

Program OPCInt.exe yang berjalan di PI Interface Node. Berlangganan item OPC dari ABB dan meneruskan nilainya ke PI Buffer/Archive.

🗄️

PI Data Archive

Menerima dan menyimpan semua time-series data dari ABB. Setiap OPC item yang dipantau = satu PI Tag di Archive.

[ WAYANG WINDU — Integration Architecture ] ABB Side PI Side ───────────────────── ──────────────────────────── ABB 800xA / MOD DCS PI Interface Node (Windows) │ │ ├─ ABB OPC DA Server ──────────▶│ OPCInt.exe (/id=1) │ ProgID: ABBGroup.OPCServer │ ↓ Exception Filter │ Items: AI.FIC101.PV │ PIBufss (disk buffer) │ AI.TIC202.PV │ ↓ Network │ DI.MOT101.RUN │ │ ... ▼ │ PI Data Archive (PISRVCISDEMO) │ ├─ WAWW.FIC101.PV (Float32) │ ├─ WAWW.TIC202.PV (Float32) │ ├─ WAWW.MOT101.RUN (Digital) │ └─ ... (ribuan tags)
ℹ️
OPC DA vs OPC UA Integrasi ABB ke PI di Wayang Windu menggunakan OPC DA (Classic) yang berbasis COM/DCOM — teknologi Windows lama namun sangat umum di DCS generasi ini. OPC DA bekerja secara subscription: Interface mendaftar ke OPC Server, dan OPC Server mengirim notifikasi setiap ada perubahan nilai (bukan polling konstan).

2.3 Informasi yang Perlu Disiapkan dari ABB

📋

Tag List ABB (I/O List)

Daftar semua tag ABB yang ingin dimonitor — nama tag, deskripsi, satuan, range min/max, dan tipe data (Analog/Digital).

🔑

OPC Server ProgID

Nama identifier OPC Server ABB, contoh: ABBGroup.OPCServer atau ABB.OPCDA.1. Dapatkan dari tim ABB atau cek via OPC client tool.

🌐

IP / Hostname ABB Node

IP address atau hostname Windows machine tempat ABB OPC DA Server berjalan (biasanya ABB Engineering Workstation).

🗂️

OPC Item Path / Browse Path

Hierarki item OPC dari ABB — bisa berupa flat list atau bertingkat, contoh: Channel1.Device1.AI.FIC101.PV.

⚠️
DCOM — Sumber Masalah Terbesar OPC DA OPC DA menggunakan COM/DCOM Windows. Firewall harus mengizinkan TCP 135 (RPC Endpoint Mapper) + dynamic DCOM ports (biasanya 1024–65535). Pastikan user account yang menjalankan OPCInt service punya DCOM Launch & Activation Permission ke OPC server ABB. Tanpa konfigurasi DCOM yang benar, interface tidak akan bisa connect ke OPC Server meski jaringan sudah tembus.
03 — Fase 1

Verifikasi OPC DA Server (ABB)

Sebelum membuat PI Tag, pastikan kamu bisa melihat dan membaca data dari OPC Server ABB. Gunakan tool OPC Client untuk browse dan test koneksi.

3.1 Browse OPC Items dari ABB

Gunakan Matrikon OPC Explorer atau tool OPC Client lainnya dari PI Interface Node untuk browse item yang tersedia di OPC Server ABB:

  1. Buka Matrikon OPC Explorer (atau OPC Quick Client) di PI Interface Node
  2. Klik Connect → pilih Remote Computer → masukkan IP/hostname ABB OPC Node
  3. Dari daftar OPC Server yang muncul, pilih OPC Server ABB (contoh: ABBGroup.OPCServer)
  4. Browse tree hierarki OPC item yang tersedia — catat nama Item ID lengkap untuk setiap tag yang ingin dipantau
  5. Tambahkan beberapa item ke group, amati apakah nilai berubah secara real-time (Quality: Good)
[ Contoh Struktur OPC Item ABB di Wayang Windu ] ABBGroup.OPCServer └── WAYANG_WINDU ├── ANALOG_INPUT │ ├── FIC101.PV ← Flow indicator controller 101, Process Value │ ├── FIC101.SP ← Setpoint │ ├── TIC202.PV ← Temperature indicator controller 202 │ ├── PIC301.PV ← Pressure indicator controller 301 │ └── LIC401.PV ← Level indicator controller 401 ├── DIGITAL_INPUT │ ├── MOT101.RUN ← Motor 101 running status │ ├── VLV201.OPEN ← Valve 201 open status │ └── PUMP301.FAULT ← Pump 301 fault alarm └── ANALOG_OUTPUT └── FIC101.OP ← Controller output %
ℹ️
Catat Full Item ID dengan Tepat Item ID OPC adalah case-sensitive dan harus persis sama saat dikonfigurasi di PI Tag (instrumenttag attribute). Salah satu karakter = tag tidak bisa baca data. Contoh full Item ID: WAYANG_WINDU.ANALOG_INPUT.FIC101.PV

3.2 Test Kualitas Data OPC

Sebelum lanjut, pastikan semua item yang ingin dipantau memiliki kualitas data yang baik:

04 — Fase 2

Konfigurasi PI OPC DA Interface

PI OPC DA Interface (OPCInt.exe) adalah bridge antara ABB OPC Server dan PI Data Archive. Interface ini dikonfigurasi via PI Interface Configuration Utility (PI ICU).

4.1 Konfigurasi via PI ICU

1

Buka PI ICU di Interface Node

Jalankan PI Interface Configuration Utility (PI ICU) dari Start Menu atau:

C:\Program Files\PIPC\ICU\PIicu.exe
2

Buat atau Pilih Interface Instance

Klik Add untuk membuat instance baru, atau pilih instance yang sudah ada untuk ABB. Pilih OPC DA sebagai interface type.

3

Isi Parameter Koneksi

Isi semua field berikut di tab General:

ParameterValueKeterangan
/hostPISRVCISDEMONama PI Data Archive server
/id1Interface ID unik (1–255). ABB = 1, nanti SETPOINT = 2
/opcserverABBGroup.OPCServerProgID OPC Server ABB (sesuaikan!)
/opchost192.168.x.xIP address ABB OPC node (jika remote)
/s5Scan interval dalam detik
/psLPoint Source — semua PI tag ABB harus punya pointsource=L
/bf1Aktifkan PIBufss buffering (zero data loss)
4

Apply & Start Service

Klik Apply, kemudian Start service. Cek status di Windows Services (services.msc) — pastikan service PI-OPCInt-1 berjalan dan statusnya Running.

# Verifikasi di PowerShell:
Get-Service "PI-OPCInt-1"
# Expected output: Running
5

Cek Log Interface

Lihat PI Interface log untuk memastikan koneksi ke OPC Server dan PI Archive berhasil:

C:\Program Files\PIPC\Interfaces\OPCInt\OPCInt1.log

# Baris yang menandakan sukses:
Connected to OPC server: ABBGroup.OPCServer
Connected to PI Server: PISRVCISDEMO
Subscribed to N OPC items successfully

4.2 PI Trust & Security

PI Interface membutuhkan PI Trust agar bisa menulis data ke PI Archive tanpa password. Konfigurasi di PI SMT:

  1. Buka PI SMT → pilih server PISRVCISDEMO
  2. Navigasi ke Security → Trusts
  3. Klik New, isi sebagai berikut:
# PI Trust untuk ABB OPC Interface:
Trust Name  : ABB_OPC_Trust
Description : Trust for ABB OPC DA Interface (Wayang Windu)
Net Mask    : 192.168.x.x    # IP Interface Node
App Name    : OPCInt
PI Identity : PIInterfaces   # Identity dengan hak write data
💡
PIInterfaces Identity Identity PIInterfaces secara default hanya punya hak Write Data (nilai) tapi tidak bisa ubah konfigurasi tag. Ini prinsip least-privilege yang benar — interface hanya bisa kirim data, tidak bisa hapus atau modifikasi tag.
05 — Fase 3

Membuat PI Tags dari ABB

Setiap OPC item yang ingin direkam ke PI Archive harus didefinisikan sebagai PI Tag. Kunci koneksinya ada di dua atribut: pointsource (harus sama dengan /ps di interface = L) dan instrumenttag (harus sama persis dengan OPC Item ID dari ABB).

🔑 Dua Atribut Kunci Linking PI Tag ke OPC Item ABB

pointsource = L → Menentukan bahwa tag ini dibaca oleh interface dengan /ps=L (yaitu interface ABB OPC DA)
instrumenttag = WAYANG_WINDU.ANALOG_INPUT.FIC101.PV → Full OPC Item ID dari ABB yang akan dibaca

5.3 Bulk via PI Builder (Excel) — Rekomendasi untuk Production

Cara paling efisien saat bekerja dengan ratusan tag dari I/O List ABB format Excel:

  1. Buka Excel, aktifkan PI Builder Add-in (tab PI Builder muncul di ribbon)
  2. Klik Build → From Template → pilih template PI Point
  3. Isi data dari I/O List ABB — setiap baris = satu PI Tag. Kolom wajib: Tag, PointType, Descriptor, EngUnits, PointSource, InstrumentTag
  4. Klik Write → PI System, pilih server PISRVCISDEMO
  5. Review hasil di panel PI Builder — baris merah = error (tagname duplikat, field wajib kosong, dsb)
  6. Perbaiki error, klik Write lagi sampai semua baris hijau (success)
💡
Mapping I/O List ABB → PI Tag Attribute
I/O List ABB (Excel) biasanya punya kolom: Tag Number, Description, Eng Unit, Range Lo, Range Hi, I/O Type.
Mappingnya ke PI: Tag Number → instrumenttag & basis tag, Description → descriptor, Eng Unit → engunits, Range Lo → zero, Range Hi − Lo → span, I/O Type AI → PointType R, DI → PointType D.
06 — Fase 4

Verifikasi Data Masuk ke Archive

Setelah PI Tag dibuat dan interface berjalan, lakukan verifikasi berlapis untuk memastikan data benar-benar mengalir dari ABB DCS sampai ke PI Archive.

Checklist Verifikasi

  • PI Interface service (PI-OPCInt-1) status Running di Windows Services
  • Interface log tidak menampilkan error koneksi ke OPC Server atau PI Archive
  • Tag di PI SMT menampilkan nilai numerik (bukan Pt Created atau Comm Fail)
  • Nilai tag berubah seiring waktu (tidak frozen) — verifikasi dengan trend 5 menit
  • Nilai tag masuk akal sesuai range engineering (tidak ada nilai di luar zero–span)
  • Data historis bisa di-query untuk rentang waktu sebelumnya
  • Tag digital menampilkan state text (Running, Open) bukan angka

6.1 Cek Nilai Tag via PI SMT

  1. Buka PI SMT → Points → Point Builder
  2. Search tag yang baru dibuat, misalnya WAWW.FIC101.PV
  3. Klik tab Snapshot → lihat nilai terkini, timestamp, dan quality
  4. Jika nilai berubah-ubah sesuai proses → ✓ Data mengalir normal

6.2 Cek Historis via PI SMT Data Archive

# Di PI SMT → Data Archive → Archive Editor:
# - Pilih tag: WAWW.FIC101.PV
# - Start time: *-10m (10 menit lalu)
# - End time:   * (sekarang)
# - Klik Search → harus ada baris data

# Atau via PI DataLink (Excel add-in):
# =PIArcVal("WAWW.FIC101.PV", "*-5m", 1)
# =PITimeDat("WAWW.FIC101.PV", "*-5m", "*", "10s")

6.3 Cek via PI Web API (Python)

import requests, json

BASE = "https://PISRVCISDEMO/piwebapi"
AUTH = ("piadmin", "password")

# Cari WebID tag yang baru dibuat
r = requests.get(f"{BASE}/points",
    params={"nameFilter": "WAWW.FIC101.PV"}, auth=AUTH, verify=False)
web_id = r.json()["Items"][0]["WebId"]
tag_name = r.json()["Items"][0]["Name"]

# Ambil current value
cv = requests.get(f"{BASE}/streams/{web_id}/value", auth=AUTH, verify=False).json()
print(f"[{tag_name}] {cv['Timestamp']}: {cv['Value']} → Quality: {cv.get('IsGood', 'N/A')}")

# Ambil historis 10 menit terakhir
hist = requests.get(f"{BASE}/streams/{web_id}/recorded",
    params={"startTime": "*-10m", "endTime": "*"}, auth=AUTH, verify=False).json()
print(f"Total recorded values (10 min): {len(hist['Items'])}")
for item in hist["Items"][-5:]:
    print(f"  {item['Timestamp']}: {item['Value']}")
💡
Berapa lama sampai data masuk? Setelah tag dibuat dan interface running, data pertama biasanya masuk dalam 5–15 detik (satu scan interval + exception filter). Kalau setelah 1 menit masih Pt Created, lanjutkan ke section Troubleshooting.

PenyebabSolusi

PenyebabSolusi

7.4 Interface Gagal Connect ke OPC Server (DCOM Error)

# Error umum di log interface:
"Failed to connect to OPC server: Access denied (0x80070005)"
"DCOM error: Class not registered (0x80040154)"

# Solusi — cek DCOM permission:
# 1. Buka dcomcnfg di Interface Node
# 2. Component Services → Computers → My Computer → Properties
# 3. Tab COM Security → Edit Limits → pastikan Allow Remote Activation

# Solusi — cek Windows Firewall:
netsh advfirewall firewall add rule name="DCOM TCP 135" ^
  protocol=TCP localport=135 dir=in action=allow

# Test koneksi OPC dari Interface Node ke ABB node:
# Buka Matrikon OPC Explorer → connect ke IP ABB node
# Jika gagal di sini → masalahnya DCOM, bukan PI
⚠️
Selalu Test OPC Koneksi Dulu Sebelum Blame PI Jika Matrikon OPC Explorer dari Interface Node bisa browse dan baca data dari ABB OPC Server dengan baik, berarti masalahnya ada di konfigurasi PI Tag atau PI Interface — bukan di jaringan atau DCOM. Pastikan test OPC client berhasil dulu sebelum troubleshoot sisi PI.

Quick Summary — Atribut Wajib untuk Tag ABB

🔑

pointsource = L

Wajib sama dengan parameter /ps=L di OPCInt. Ini yang menentukan interface mana yang akan membaca tag ini.

🔗

instrumenttag = OPC Item ID

Harus sama persis (case-sensitive) dengan Item ID di OPC Server ABB. Ini yang menentukan nilai dari mana yang akan disimpan ke tag.

📊

pointtype = R (Float32)

Untuk semua sensor analog dari ABB (AI). Gunakan D untuk digital input (valve, motor status).

⚙️

compressing = 1

Selalu aktifkan compression untuk tag analog. Menghemat storage hingga 10x tanpa kehilangan akurasi data.
