💡
Kenapa SCADA02 (VC-8000) jauh lebih lama dari SCADA01 (ABB)? Ada 3 alasan utama:
① Disk SCADA02 jauh lebih besar — ~3,600 GB vs hanya 207 GB di SCADA01 (17× lebih besar).
② Compression VC-8000 lebih agresif — SETPOINT compressing=On (4×), sedangkan ABB compressing=Off (1×). Data VC-8000 dikompresi 4× lebih kecil.
③ Exception VC-8000 juga efektif — meski vibrasi berubah terus, SETPOINT excmin=10 detik membatasi kiriman ke PI maksimal 1 kali per 10 detik.
Intinya: disk SCADA02 oversized untuk kebutuhan VC-8000 saat ini — artinya ada ruang besar untuk ekspansi tag di masa depan.

Glosarium Istilah PI System
Klik untuk buka ▼
🏷️
PI Tag
Satu "titik pengukuran" yang disimpan di PI System. Setiap sensor punya satu tag unik dengan nama seperti A1_PIT0610 atau UNIT#1.3.1.Direct.0.
💡 Ibarat nomor rekening bank — setiap sensor punya "rekening" sendiri untuk menyimpan riwayat datanya
🔍
Exception Reporting
Filter pertama PI — data sensor tidak disimpan jika nilainya tidak berubah cukup signifikan dari sebelumnya. Diatur oleh parameter excdev (batas perubahan minimum).
💡 Seperti laporan cuaca: tidak perlu tulis "masih 30°C" setiap menit jika suhu tidak berubah. Baru tulis jika berubah lebih dari 1°C.
📐
Swingdoor Compression
Filter kedua PI — dari data yang lolos Exception, hanya titik yang "penting" untuk merekonstruksi tren yang disimpan. Titik yang bisa diprediksi dari garis lurus tidak disimpan.
💡 Untuk menggambar jalan lurus, cukup tandai titik awal dan akhir — tidak perlu tandai setiap meter di tengah jalan.
⚡
Snapshot
Memori PI yang menyimpan nilai terbaru setiap tag — inilah yang ditampilkan di PI Vision secara real-time. Snapshot tidak tersimpan permanen, hilang jika server restart.
💡 Seperti papan tulis digital — menampilkan nilai "saat ini", bukan riwayat. Kalau server restart, papan tulis ini terhapus.
🗄️
Archive File (.arc)
File penyimpanan permanen data historis PI di hard disk. Setiap file berukuran tetap (10,240 MB di Wayang Windu). Ketika penuh, PI membuat file baru secara otomatis.
💡 Seperti buku log fisik — setiap buku punya kapasitas halaman. Kalau penuh, ambil buku baru. Buku lama tetap ada dan bisa dibaca.
📊
Scalar Tag
Tag yang menyimpan satu angka per pembacaan — misalnya tekanan: 50.2 bar, temperatur: 85.3°C, kecepatan: 3000 rpm. Sebagian besar tag proses ABB dan parameter utama VC-8000 adalah scalar.
💡 Seperti termometer digital — satu angka per pembacaan. 215 tag ABB dan 181 tag scalar VC-8000 di Wayang Windu semuanya scalar.
〰️
Waveform Tag
Tag khusus VC-8000 yang menyimpan seluruh bentuk gelombang getaran — ribuan angka sekaligus per capture. Digunakan untuk analisis frekuensi mendalam (FFT spectrum).
💡 Seperti rekaman audio vs satu angka volume. Waveform = rekaman lengkap getaran, scalar = hanya nilai puncaknya saja. Satu capture waveform = 128× lebih besar dari scalar.
📡
OPC DA (ABB DCS → PI)
Protokol komunikasi standar industri yang digunakan ABB DCS untuk mengirim data sensor ke PI System. PI Interface (OPCInt) membaca nilai dari ABB setiap 1 detik via OPC DA.
💡 OPC DA = "bahasa" yang digunakan ABB dan PI untuk saling bicara. Seperti adapter yang memungkinkan dua perangkat berbeda terhubung.
📳
SETPOINT Connector
Program khusus dari Brüel & Kjær Vibro yang menghubungkan VC-8000 ke PI System. Berbeda dari OPC DA, SETPOINT menggunakan koneksi TCP langsung ke port 8001 di VC-8000.
💡 Versi "kabel khusus" antara VC-8000 dan PI — tidak pakai OPC seperti ABB, tapi protokol tersendiri milik Brüel & Kjær Vibro.
🌳
PI AF (Asset Framework)
Fitur PI yang mengorganisir tag menjadi hierarki aset — seperti UNIT#1 → Turbine → Shaft Vibration 1X. Memudahkan operator menavigasi data tanpa harus hafal nama tag teknis.
💡 Seperti folder di komputer — daripada cari file bernama "A1_YIY0405A", bisa klik: Pabrik → Unit 1 → Turbin → Getaran Poros Depan.
💾
Overhead
Ruang disk ekstra yang dibutuhkan untuk file pendukung — index pencarian (.idx), file anotasi, buffer sistem operasi. Data sebenarnya lebih kecil dari total ruang yang dipakai.
💡 Seperti buku yang butuh daftar isi dan cover — "overhead" dari sebuah buku. Data = isi buku, overhead = daftar isi + sampul + halaman kosong.
🔄
Backup Incremental
Metode backup yang hanya menyalin data BARU sejak backup terakhir — lebih cepat dan hemat dibanding full backup. Di Wayang Windu berjalan otomatis setiap hari pukul 03:15 AM.
💡 Seperti mencatat di buku harian — setiap hari hanya tulis yang baru, bukan menulis ulang semua cerita dari awal.
🛡️
RAID 1
Konfigurasi dua hard disk yang saling mencerminkan — setiap data ditulis ke dua disk sekaligus. Jika satu disk rusak, data tetap aman di disk yang lain. Wayang Windu: 2 × 3.84 TB RAID1.
💡 Seperti menyimpan dokumen penting di dua tempat berbeda sekaligus — kalau satu hilang, masih ada salinannya.
📦
Retensi Data
Berapa lama data historis bisa disimpan sebelum disk penuh. Dihitung dari kapasitas disk yang tersedia dibagi dengan net data rate harian setelah semua filter.
💡 Seperti umur flashdisk — kalau kapasitas 16 GB dan kamu simpan 1 GB per hari, flashdisk tahan 16 hari sebelum penuh.
🔢
Bytes per Event (BPE)
Ukuran satu titik data yang disimpan PI: 8 byte waktu + 4 byte nilai angka + 4 byte kualitas sinyal = 16 byte total. Angka ini sama untuk semua tag scalar standar PI.
💡 16 byte ≈ ukuran 16 karakter teks. Sangat kecil — tapi kalau jutaan per hari dari ratusan tag, totalnya besar.
⏱️
Scan Rate
Interval waktu antar pembacaan sensor. Scan rate 1 detik = PI membaca nilai sensor setiap 1 detik = 86.400 pembacaan per tag per hari sebelum filter exception.
💡 Seperti berapa sering kamu melihat jam — setiap detik vs setiap menit. Semakin sering, semakin banyak data, semakin butuh banyak storage.
