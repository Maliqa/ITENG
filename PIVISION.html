01 — Overview

Apa itu PI Vision?

PI Vision adalah web-based visualization tool dari AVEVA OSIsoft yang memungkinkan operator, engineer, dan manajemen untuk membuat dan melihat dashboard operasional secara real-time langsung dari browser — tanpa perlu install aplikasi klien apapun.

PI Vision berjalan di atas IIS (Internet Information Services) di Windows Server dan mengakses data dari PI Data Archive serta PI Asset Framework (PI AF) secara langsung.

🌐

Web-Based

Akses dari browser mana saja — Chrome, Edge, Firefox. Tidak perlu install klien. Mendukung desktop, tablet, dan mobile.

⚡

Real-Time

Data di-refresh otomatis dari PI Data Archive. Interval refresh bisa dikonfigurasi per display (default 5 detik).

🎨

No-Code Builder

Drag-and-drop symbol builder. Tidak perlu coding untuk membuat dashboard operasional yang powerful.

🌳

AF Integration

Terintegrasi langsung dengan PI AF — browse asset hierarchy untuk pilih data, bukan harus ketik nama tag manual.

🔒

Role-Based Access

Permission per display dan per user/group. Operator hanya bisa lihat, Engineer bisa edit, Admin bisa manage semua.

🔗

Embeddable

Display bisa di-embed ke aplikasi lain, portal web, atau ditampilkan di monitor control room via URL langsung.

03 — Display Builder

Membuat Display & Dashboard

3.1 Membuat Display Baru dari Scratch

1

Buka PI Vision & Klik New Display

Buka browser → https://PISRVCISDEMO/PIVision → Login → Klik tombol + New Display di halaman Home.

2

Set Canvas Size & Background

Klik area kosong canvas → panel kanan akan menampilkan Display Properties. Atur Width, Height (contoh: 1920×1080 untuk full HD monitor control room), dan warna background.

3

Pilih Asset Context (Opsional tapi Disarankan)

Klik Asset di toolbar atas → Browse AF hierarchy → pilih element yang akan jadi konteks utama display (misal: Compressor-01). Ini memungkinkan display dibuat generic dan bisa di-swap assetnya saat viewing.

4

Tambahkan Symbol

Klik Symbols di toolbar kiri → pilih jenis symbol (Trend, Value, Gauge, dll) → klik di canvas untuk menempatkannya. Atau drag attribute dari Asset Browser langsung ke canvas — PI Vision otomatis membuat symbol yang sesuai.

5

Link Data ke Symbol

Klik symbol di canvas → panel kanan tampilkan Data Sources → klik + Add → search nama PI Tag atau browse AF attribute. Bisa juga drag dari Asset Browser ke symbol yang sudah ada.

6

Save & Share Display

Klik Save (Ctrl+S) → beri nama display → pilih lokasi penyimpanan: My Displays (hanya kamu) atau folder Shared (semua user bisa lihat). Set permission yang sesuai.

3.2 Tips Konfigurasi Symbol

💡
Cara Tercepat Tambah Data: Di Asset Browser (sidebar kiri), navigate ke AF element → drag attribute langsung ke canvas. PI Vision otomatis menentukan symbol type yang paling cocok (angka = Value symbol, array time-series = Trend symbol, dll).

3.4 Tips Layout & Formatting

  • Gunakan Align tools (klik kanan → Align) untuk merapikan posisi symbol secara presisi
  • Gunakan Group (Ctrl+G) untuk mengelompokkan symbol yang berkaitan agar bisa dipindah bersama
  • Tambahkan Text label di atas setiap kelompok symbol untuk header section
  • Gunakan Shape/Rectangle dengan background color berbeda untuk membagi area display secara visual
  • Set Refresh Rate per display sesuai kebutuhan — dashboard vibrasi bisa 2 detik, overview plant cukup 10 detik

05 — Multi-State

Multi-State Symbol

5.1 Konsep Multi-State

Multi-State adalah fitur PI Vision yang memungkinkan symbol berubah tampilan (warna, ukuran, ikon, teks) secara otomatis berdasarkan nilai data yang sedang dibaca. Ini adalah cara membuat display yang informatif — operator bisa langsung tahu status equipment hanya dari warna tanpa harus baca angka.

🎨

Visual State Mapping

Setiap rentang nilai atau state tertentu dipetakan ke tampilan visual berbeda — warna latar, warna teks, ikon, atau animasi berkedip untuk kondisi kritis.

🔢

Bisa untuk Tag Numerik & Digital

Bekerja untuk tag Float (berdasarkan range nilai) maupun tag Digital (berdasarkan state text seperti Running/Stopped). Juga bisa berbasis AF attribute.

5.2 Konfigurasi Multi-State Step-by-Step

  1. Di Mode Edit, klik symbol yang ingin dikonfigurasi (Value, Status Indicator, atau Shape)
  2. Di panel kanan, scroll ke bagian Format → klik Multi-State
  3. Klik + Add State untuk menambah kondisi pertama
  4. Pilih Data Source yang digunakan untuk evaluasi kondisi (bisa sama atau beda dengan data yang ditampilkan)
  5. Tentukan kondisi: pilih operator (=, >, <, between) dan nilai threshold
  6. Pilih visual yang berubah: Fill Color, Text Color, Blink (kedip), atau Icon
  7. Ulangi untuk setiap state yang dibutuhkan (Normal, Alert, Danger)
  8. Klik Done → Save display (Ctrl+S)

5.3 Contoh: Motor Status (Digital Tag)

Motor WAWW.MOT101.RUN bertipe Digital dengan state set MotorState. Konfigurasi Multi-State:

06 — Risk Level

Risk Level Visualization

Normal / Alert / Danger

6.1 Konsep Risk Level

Risk Level visualization adalah penerapan Multi-State yang lebih terstruktur — menggunakan tiga zona standar industri yang dipetakan ke nilai threshold engineering. Ini sangat umum dipakai untuk parameter vibrasi, temperature, dan tekanan.

✅
Normal
RMS < 3.0 mm/s
Operasi normal, tidak perlu tindakan
⚡
Alert
3.0 ≤ RMS < 7.1 mm/s
Pantau lebih ketat, jadwalkan inspeksi
🚨
Danger
RMS ≥ 7.1 mm/s
Tindakan segera! Pertimbangkan shutdown
ℹ️
Referensi Threshold Vibrasi — ISO 10816
Threshold di atas mengacu pada standar ISO 10816 untuk mesin rotating. Nilai aktual untuk Wayang Windu bisa berbeda — selalu konfirmasi dengan datasheet equipment (VC-8000 / STI CMCP601) dan rekomendasi OEM.

6.2 Membuat Enumeration Set di PI AF

Sebelum dikonfigurasi di PI Vision, buat Enumeration Set di PI AF yang mendefinisikan tiga level risk. Ini memungkinkan AF Analytics menghitung risk level dan menyimpannya sebagai Digital state.

  1. Buka PI System Explorer (PSE) → pilih AF Database PLANT_DEMO
  2. Klik menu Elements → Enumeration Sets → New Enumeration Set
  3. Beri nama: VibrationRiskLevel
  4. Tambahkan tiga entry: 0 = Normal, 1 = Alert, 2 = Danger
  5. Klik Check In untuk menyimpan
/* Enumeration Set: VibrationRiskLevel */
0 → Normal   // RMS < 3.0 mm/s
1 → Alert    // 3.0 ≤ RMS < 7.1 mm/s
2 → Danger   // RMS ≥ 7.1 mm/s

/* AF Analytics Expression untuk hitung risk level: */
If TagVal('Vibration_RMS') >= 7.1 Then 'Danger'
Else If TagVal('Vibration_RMS') >= 3.0 Then 'Alert'
Else 'Normal'

/* Output: PI Tag Digital dengan DigitalSet = VibrationRiskLevel */
/* Contoh tag: WAWW.COMP01.VIB.RISKLEVEL (tipe: Digital) */

6.3 Konfigurasi Risk Level di PI Vision

Setelah PI Tag risk level dibuat dan AF Analytics berjalan, konfigurasi visualisasinya di PI Vision:

1

Tambahkan Status Indicator Symbol

Di Mode Edit → Symbols → pilih Status Indicator (lingkaran atau kotak) → klik di canvas.

2

Link ke Risk Level Tag

Di panel kanan → Data Sources → Add → search tag WAWW.COMP01.VIB.RISKLEVEL (Digital tag output dari AF Analytics).

3

Konfigurasi Multi-State

Format → Multi-State → tambah 3 state:

StateKondisiFill ColorText ColorBlink
Normaltag = Normal● #00875a#fffTidak
Alerttag = Alert● #d97706#fffTidak
Dangertag = Danger● #dc2626#fffYa
4

Tambahkan Value Symbol di Sebelahnya

Tambah symbol Value di sebelah Status Indicator → link ke WAWW.COMP01.VIB.RMS → operator bisa sekaligus lihat warna status DAN angka aktualnya.

6.4 Animated Gauge Widget dengan Risk Zone

Untuk tampilan yang lebih visual dan intuitif, gunakan Gauge symbol dengan zone color berbeda:

  1. Tambah Gauge symbol → link ke WAWW.COMP01.VIB.RMS
  2. Di Format panel → set Minimum: 0, Maximum: 15 (mm/s)
  3. Aktifkan Ranges → klik Add Range
  4. Range 1: Start=0, End=3.0, Color=#00875a (Normal)
  5. Range 2: Start=3.0, End=7.1, Color=#d97706 (Alert)
  6. Range 3: Start=7.1, End=15, Color=#dc2626 (Danger)
  7. Aktifkan Show Value agar angka numerik juga tampil di tengah gauge
  8. Aktifkan Multi-State pada gauge needle agar ikut berubah warna sesuai zone aktif
💡
Kombinasi yang Optimal untuk Dashboard Vibrasi: Gunakan tiga elemen berdampingan per equipment — (1) Gauge dengan zone color untuk kesan visual langsung, (2) Value untuk angka presisi, (3) Trend kecil untuk historis 8 jam terakhir. Operator mendapat semua informasi yang dibutuhkan dalam satu pandang.

7.3 Overlay Vibrasi + Proses dalam Satu Display

Ini adalah dashboard paling powerful — menampilkan data vibrasi dari SETPOINT dan data proses dari ABB dalam satu layar untuk analisis korelasi:

1

Bagi Canvas Menjadi Dua Area

Gunakan Rectangle symbol sebagai background section. Bagian atas = vibrasi (background abu muda), bagian bawah = proses (background biru muda). Tambahkan label teks sebagai header tiap section.

2

Tambahkan Multi-Tag Trend

Satu Trend symbol bisa menampung multiple tag. Buat satu trend besar yang menampilkan: VIB.RMS (axis kiri, mm/s) + TEMP.IN (axis kanan, °C) + SUCT.PRESS (axis kanan, barg). Operator bisa langsung korelasikan naik-turun vibrasi dengan kondisi proses.

3

Set Dual Y-Axis

Klik Trend → Format → di setiap data source, pilih apakah menggunakan Left Axis atau Right Axis. Vibrasi (mm/s) di kiri, suhu/tekanan di kanan. Set range axis masing-masing agar proporsional.

4

Tambahkan Reference Lines di Trend

Di Trend → Format → Reference Lines → tambahkan garis horizontal di nilai 3.0 dan 7.1 mm/s (alert dan danger threshold vibrasi). Beri label dan warna sesuai.

08 — Security

User Permission & Security

PI Vision menggunakan Windows Authentication dan integrasi dengan PI Identity untuk mengontrol siapa yang bisa melihat dan mengedit display.

RoleHak AksesContoh User
PI Vision AdministratorFull control — manage server, user, semua displayAdmin PIMS, IT administrator
PI Vision CreatorBuat dan edit display milik sendiri + share ke folder publicEngineer, Implementor
PI Vision ReaderHanya bisa melihat display yang di-share. Tidak bisa edit.Operator lapangan, manajemen
No AccessTidak bisa login ke PI VisionUser yang tidak terdaftar di PI Identity

Cara Mengatur Permission Display

  1. Buka display yang ingin diatur → klik ikon ⋯ (More) di pojok kanan atas
  2. Pilih Security → panel permission muncul
  3. Klik + Add → cari Windows User atau Group (misalnya DOMAIN\Operators)
  4. Pilih level: Read (hanya lihat) atau Write (bisa edit)
  5. Klik Save
⚠️
Akses Data Dikontrol PI Server, bukan PI Vision! PI Vision hanya mengontrol siapa yang bisa lihat/edit display. Apakah user tersebut bisa membaca data dari PI Archive diatur oleh PI Identity mapping dan tag security (datasecurity attribute) di PI Server. Dua layer keamanan ini independen.

10 — Troubleshooting

Troubleshooting PI Vision

⬜ PI Vision tidak bisa dibuka — halaman putih atau blank 🔥 Paling Sering
⚠️ Penyebab

IIS mengalami masalah — bisa karena service crash, corrupt application pool, atau Windows update yang mengganggu konfigurasi IIS.

✅ Solusi Bertahap
  1. Reset IIS: buka Command Prompt as Administrator → iisreset /noforce
  2. Coba buka dari browser berbeda dan clear cache browser (Ctrl+Shift+Del)
  3. Cek PI Vision Application Pool di IIS Manager — pastikan status Started
  4. Cek Windows Event Viewer → Application log untuk error spesifik terkait PIVision atau W3SVC
  5. Jika error 500 atau corrupt: uninstall PI Vision → reinstall IIS → reinstall PI Vision
# Langkah 1 — Quick IIS reset:
iisreset /noforce

# Langkah 5 — Reinstall IIS jika corrupt:
Get-WindowsFeature "Web-*" | Where-Object {$_.InstallState -eq "Installed"} | Uninstall-WindowsFeature
Install-WindowsFeature -Name Web-Server -IncludeManagementTools -IncludeAllSubFeature
# Lalu reinstall PI Vision dari installer
🔴 Symbol menampilkan "No Data" atau "No Values" padahal tag ada datanya 🔥 Sering
⚠️ Penyebab Umum
  • User tidak punya hak akses baca ke tag tersebut (PI data security)
  • Time range di display tidak ada datanya (misal minta data 5 tahun lalu tapi archive tidak ada)
  • PI Server connection dari PI Vision terputus
  • Tag baru dibuat tapi belum ada data masuk (Pt Created)
✅ Diagnosa
  1. Cek di PI SMT apakah tag tersebut ada data di Snapshot — jika ada data di SMT tapi tidak di Vision, kemungkinan masalah permission.
  2. Cek datasecurity atribut tag — pastikan user/identity yang dipakai PI Vision punya hak Read.
  3. Cek PI Vision Admin → PI Server Connection → pastikan status Connected dan tidak ada error.
  4. Adjust time range di display — coba set ke *-1h (1 jam terakhir) untuk test.
⚠️ Multi-State tidak bekerja — warna tidak berubah sesuai kondisi Konfigurasi
⚠️ Penyebab
  • Kondisi Multi-State dievaluasi terhadap data source yang salah
  • Tipe operator salah — misalnya pakai = untuk nilai Float (harus pakai between)
  • Urutan state tidak benar — PI Vision mengevaluasi dari atas ke bawah, state pertama yang cocok yang dipakai
  • Untuk Digital tag: nilai kondisi harus persis nama state text-nya (case-sensitive!)
✅ Solusi
  1. Klik symbol → Format → Multi-State → pastikan Data Source yang dipakai untuk evaluasi sudah benar.
  2. Untuk tag Float: gunakan operator between dengan nilai min dan max yang tepat.
  3. Untuk Digital tag: pastikan nilai kondisi adalah exact state name dari Digital State Set (Running bukan running).
  4. Ubah urutan state — kondisi paling spesifik (misalnya Danger) taruh di posisi paling atas.
🐌 Display loading sangat lambat atau trend tidak muncul Performance
⚠️ Penyebab
  • Time range terlalu panjang dengan terlalu banyak tag dalam satu trend (misal 1 tahun × 50 tag)
  • Terlalu banyak symbol dalam satu display (biasanya >100 symbol)
  • IIS server resource kurang (RAM/CPU) — biasanya karena banyak user buka secara bersamaan
  • Refresh rate terlalu pendek (1 detik) dengan banyak data source
✅ Solusi
  1. Batasi jumlah tag per Trend symbol maksimal 10–15 untuk performa optimal.
  2. Gunakan mode Interpolated dengan interval lebih besar (misal 1 menit) untuk trend dengan range panjang.
  3. Bagi display besar menjadi beberapa halaman yang lebih kecil dan hubungkan dengan Link symbol.
  4. Naikkan refresh rate dari 2 detik ke 5 atau 10 detik untuk display overview yang tidak perlu update sangat cepat.
🔒 User tidak bisa login atau muncul "Unauthorized" saat buka display Permission
✅ Langkah Diagnosa
  1. Cek apakah user bisa login ke PI Vision sama sekali — jika tidak bisa, berarti Windows account belum di-assign ke PI Vision role. Hubungi PI Vision Administrator.
  2. Jika bisa login tapi tidak bisa buka display tertentu: display tersebut private atau belum di-share. Minta pemilik display untuk share ke user/group yang sesuai.
  3. Jika bisa buka display tapi data tidak muncul: periksa PI data security — user tidak punya hak baca ke PI tag yang ada di display tersebut.

📱 PI Vision tidak tampil dengan baik di tablet atau mobile Display
✅ Tips
  • Gunakan Responsive layout saat membuat display — hindari posisi fixed pixel yang terlalu presisi
  • Buat display terpisah khusus untuk mobile dengan canvas size 375×812 (iPhone) atau 768×1024 (iPad)
  • Kurangi jumlah symbol per display untuk layar kecil
  • Gunakan parameter URL hidenav=true untuk maksimalkan ruang layar
