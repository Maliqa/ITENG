Rekomendasi Resmi AVEVA: Jangan Install PI ProcessBook

AVEVA secara resmi menyatakan bahwa PI ProcessBook telah "officially retired" dan merekomendasikan semua pengguna beralih ke PI Vision sebagai satu-satunya jalur migrasi yang didukung. Bukan sekadar deprecated — ini full retirement.

// 02 · Risiko Jika Tetap Install

Risk Assessment: PI ProcessBook di Environment Baru

Jika tetap memilih untuk menginstall PI ProcessBook (misalnya dari installer lama), berikut adalah risiko-risiko yang harus dipahami secara teknis:

🔓
● Critical
Unpatched Security Vulnerabilities
Sejak Des 2022, tidak ada security patch. CVE yang ditemukan setelah tanggal tersebut pada komponen ActiveX, VBA runtime, dan COM objects yang digunakan ProcessBook tidak akan pernah ditambal. Attack surface terbuka permanen.
🧩
● Critical
Dependency Hell: ActiveX & COM
ProcessBook bergantung pada teknologi ActiveX dan COM/OLE yang dijalankan lewat Internet Explorer engine. IE sudah resmi retired Juni 2022. Windows 11 / Windows Server 2022+ tidak lagi support fully IE-based COM rendering.
🧱
◆ High
Incompatibility dengan OS Terbaru
ProcessBook dirancang untuk Windows XP–Windows 10 era lama. Instalasi di Windows Server 2019/2022 atau Windows 11 dapat mengalami kegagalan rendering, crash, atau fitur yang silent-fail tanpa error message.
🚫
◆ High
No Support & No Download
AVEVA Customer Portal sudah menutup akses download resmi. Menggunakan installer dari sumber tidak resmi melanggar lisensi dan meningkatkan risiko supply chain attack (installer yang telah dimodifikasi).
⚡
◆ High
Konflik dengan PI Vision di Server yang Sama
ProcessBook menggunakan port dan registry key yang bisa conflict dengan PI Vision (IIS-based). Menginstall keduanya di server yang sama berisiko menyebabkan PI Vision gagal berfungsi normal.
🔧
▸ Medium
Dead-End Migration Path
Semakin lama menggunakan ProcessBook, semakin besar technical debt yang menumpuk terutama jika ada custom VBA scripting. Migrasi ke PI Vision belakangan akan lebih kompleks dan costly.

// 04 · Analogi Sederhana

Kalau Bingung, Baca Ini

Analogi 1 — Warung vs Restoran Online
PI ProcessBook itu seperti warung makan jadul yang sudah tutup. Dulu enak, dulu bisa makan di sana, tapi sekarang sudah tutup permanen, kuncinya sudah diserahkan, dan pemiliknya sudah pindah buka restoran baru (PI Vision).

Kamu masih bisa masuk lewat jendela yang rusak (installer bajakan/lama) dan masak sendiri di dapur yang ditinggal, tapi tidak ada yang jaga, tidak ada yang bersihkan, tidak ada yang tahu kalau ada tikus masuk.
ProcessBook =
Warung tutup yang kamu paksa buka sendiri
No Security Patch =
Pintu rusak yang tidak ada yang mau betulkan
PI Vision =
Restoran baru, bisa order lewat HP dari mana saja
Migrasi =
Pindah ke restoran baru, menu sama, tampilan lebih bagus
Analogi 2 — Windows XP di 2025
Menginstall PI ProcessBook di server baru itu seperti menginstall Windows XP di laptop gaming terbaru. Bisa jalan? Mungkin. Tapi:

— Driver tidak support penuh
— Tidak ada update keamanan sejak 2014
— Hacker sudah hafal semua lubangnya
— Tidak ada yang mau bantu kalau rusak

Bedanya, Windows XP setidaknya bisa kamu download legal. PI ProcessBook bahkan sudah dihapus dari portal resmi AVEVA.
Analogi 3 — Untuk Tim Petronas
Bayangkan kamu pasang sensor transmitter lama yang sudah discontinued di fasilitas produksi baru. Transmitter bisa baca tekanan dan suhu, tapi spare part tidak tersedia, kalibrasi tidak bisa dilakukan karena vendor sudah tutup, dan kalau bocor tidak ada warranty coverage.

Regulator (audit ISA-99/IEC 62443) akan langsung flag ini sebagai non-compliant component. Itulah posisi PI ProcessBook di stack PI System modern — bukan sekadar "kurang bagus", tapi actively non-compliant dari sisi security baseline.
// 05 · Kesimpulan

Verdict

Jangan Install. Gunakan PI Vision.

AVEVA telah resmi menutup PI ProcessBook per 31 Desember 2024. Tidak ada security patch, tidak ada support, tidak ada download resmi. Menginstall di environment baru (terutama production atau lab yang mensimulasikan production) adalah technical debt yang tidak worth it — terutama di lingkungan regulated seperti Oil & Gas Petronas yang memerlukan kepatuhan terhadap ISA-99 / IEC 62443.

PI Vision adalah satu-satunya jalur forward yang didukung AVEVA.

Scenario Rekomendasi Alasan
Lab / Simulasi baru ✓ Pakai PI Vision Simulasi harusnya cerminkan production stack yang valid
